## 개요

Campaign 통계는 Campaign이 얼마나 잘 되고 있는지 보여줍니다. 얼마나 많은 메시지가 발송되었는지, 얼마나 많은 사람이 클릭했는지, 얼마나 많이 전환되었는지 등의 수치입니다.

## 통계 작동 방식

### 2계층 시스템

Notifly는 두 가지 방법으로 통계를 가져옵니다:

1. **사전 집계 데이터** (기본)
   - 숫자가 매시간 미리 계산됨
   - 빠른 로딩
   - 시간별 버킷으로 구성

2. **대체 데이터** (백업)
   - 원시 이벤트 데이터를 요청 시 계산
   - 사전 집계 데이터가 없을 때 사용
   - 느리지만 항상 사용 가능

### 데이터 흐름

1. 이벤트 발생 (발송, 클릭, 오픈 등)
2. 이벤트가 시간별 버킷으로 수집됨
3. 버킷이 통계로 집계됨
4. 통계가 대시보드에 표시됨

## 주요 지표

### 표준 지표

| 지표 | 의미 |
|------|------|
| `send_success` | 메시지가 성공적으로 발송됨 |
| `push_click` | 사용자가 푸시 알림을 탭함 |
| `email_open` | 사용자가 이메일을 열음 |
| `email_click` | 사용자가 이메일의 링크를 클릭함 |
| `in_app_message_show` | 인앱 메시지가 표시됨 |
| `in_web_message_show` | 인웹 메시지가 표시됨 |
| `main_button_click` | 사용자가 메인 CTA 버튼을 클릭함 |
| `link_open` | 사용자가 링크를 클릭함 |

### 전환 지표

| 유형 | 의미 |
|------|------|
| `total_count_conversion` | 전환 이벤트 수 |
| `sales_conversion` | 총 판매 금액 (합계) |
| `total_sum_conversion` | 속성의 총 값 (합계) |

## lastCollectedAt

`lastCollectedAt` 타임스탬프는 통계가 마지막으로 업데이트된 시점을 보여줍니다.

- 형식: ISO 8601 타임스탬프 (예: `2024-01-15T09:00:00.000Z`)
- 새 데이터가 처리될 때마다 매시간 업데이트
- null이면 데이터가 대체(원시 이벤트)에서 가져온 것

### 의미

| lastCollectedAt | 의미 |
|-----------------|------|
| 최근 타임스탬프 | 데이터가 사전 집계 시스템에서 가져온 것 |
| Null/누락 | 데이터가 대체에서 가져온 것 (더 느릴 수 있음) |
| 오래된 타임스탬프 | 집계가 지연되었을 수 있음 |

## 시간대 처리

- **저장**: UTC (협정 세계시)
- **표시**: KST (한국 표준시, UTC+9)
- **날짜 경계**: 한국 사용자를 위해 KST로 계산

### 중요한 이유

월요일 오후 10시 KST에 보낸 Campaign:
- 월요일 오후 1시 UTC로 저장됨
- 한국 대시보드에서 월요일로 표시됨
- 한국 날짜 기준으로 통계가 그룹화됨

## 캐싱

통계 쿼리는 성능 향상을 위해 **60초** 동안 캐시됩니다.

이는 다음을 의미합니다:
- 최근 변경 사항이 나타나기까지 최대 60초 소요
- 즉시 새로고침해도 같은 숫자가 표시될 수 있음
- 1분 기다린 후 새로고침하면 업데이트된 내용 표시

## 변형 (A/B 테스트)

A/B 테스트 Campaign의 경우 통계가 변형별로 추적됩니다:
- 각 변형에 자체 숫자가 있음
- 변형 간 성과 비교 가능
- 총 Campaign 통계는 모든 변형을 합산

## 집계 작동 방식 (백엔드)

백그라운드에서 Notifly는 매시간 실행되는 예약된 작업을 통해 통계를 처리합니다. 자세한 문서는 [집계 개요](/ko/knowledge-base/campaigns/aggregation/index)를 참조하세요.

### 4가지 유형의 통계 작업

| 작업 | 역할 | 세부 정보 |
|------|------|-----------|
| Campaign 메시지 지표 | 발송, 클릭, 오픈 횟수 계산 | [메시지 이벤트](/ko/knowledge-base/campaigns/aggregation/message-events) |
| 전환 추적 | 구매, 가입 추적 | [전환 추적](/ko/knowledge-base/campaigns/aggregation/conversion-tracking) |
| User Journey 통계 | Journey 노드 성과 추적 | [Journey 집계](/ko/knowledge-base/journeys/aggregation) |
| Experiment 결과 | A/B 테스트 스코어카드 계산 | [스코어카드 계산](/ko/knowledge-base/experiments/scorecard-calculation) |

### 처리 주기

- 작업이 일정에 따라 **매시간** 실행됨 (Experiment는 3시간마다)
- S3에서 원시 이벤트 수집
- 집계된 결과가 PostgreSQL에 저장됨
- `lastCollectedAt` 타임스탬프가 각 성공적인 실행 후 업데이트됨

### 2계층 데이터 저장

| 계층 | 저장소 | 목적 |
|------|--------|------|
| 원시 이벤트 | S3 (데이터 레이크) | 장기 저장, 대체 쿼리 |
| 집계된 통계 | PostgreSQL | 빠른 대시보드 쿼리 |

### 차원 기반 분석

통계는 사용자 또는 디바이스 속성별로 분석할 수 있습니다. 자세한 내용은 [차원](/ko/knowledge-base/campaigns/aggregation/dimensions)을 참조하세요.
