## 개요

Campaign을 시작하기 전에 Notifly가 모든 것이 올바르게 설정되었는지 확인합니다. Campaign이 따라야 하는 규칙입니다.

## 공통 유효성 검사

모든 Campaign에 적용되는 규칙:

| 필드 | 규칙 |
|------|------|
| Campaign 이름 | 필수, 1-50자 |
| 설명 | 선택사항, 최대 150자 |
| 전환 이벤트 | 최대 4개, 중복 불가, `notifly__`로 시작 불가 |
| 분석 차원 | 최대 6개, 중복 불가 |
| 윈도우 일수 | 1-14일 |

## Experiment Campaign 규칙

A/B 테스트 Campaign에는 추가 규칙이 있습니다:

| 규칙 | 설명 |
|------|------|
| 전환 이벤트 | 최소 1개 필수 (어떤 변형이 이기는지 측정하기 위해) |
| 그룹 | 2-5개 그룹 필요 |
| 비율 | 모든 그룹 비율의 합이 정확히 100%여야 함 |
| 대조군 | 최대 1개의 대조군 허용 |
| 그룹 이름 | 고유해야 함 (중복 불가) |
| Segment 모드 | AI Assistant 모드 사용 불가 |

## 타이밍 유효성 검사

| 타이밍 유형 | 규칙 |
|-------------|------|
| Scheduled | 미래에 예약 가능한 시간이 최소 하나 있어야 함 |
| Recurring | 종료 날짜가 시작 날짜 이후여야 함 |

## Segment 유효성 검사

| 규칙 | 적용 대상 |
|------|-----------|
| Segment 조건에 개인화 변수 사용 불가 | 인앱 및 인웹 메시지 채널 |
| 사용자 ID 기반이 아닌 CSV Segment: csv 변수만 허용 | 이메일, SMS, 카카오, LINE 채널 |

## 발신자 정보 요구 사항

일부 채널은 프로젝트에 발신자 정보가 구성되어 있어야 합니다:

| 채널 | 필요한 구성 |
|------|-------------|
| 푸시 알림 | FCM 서버 키 또는 FCM 서비스 계정 |
| 웹 푸시 | 웹 푸시 발신자 정보 |
| 이메일 | 이메일 메시지 발신자 정보 |
| 문자 메시지 (SMS) | 문자 메시지 발신자 정보 |
| 카카오 (모든 유형) | 카카오 발신자 정보 |
| LINE | LINE 채널 정보 |

## Experiment 그룹 유효성 검사

A/B 테스트 Campaign의 경우 각 그룹은 다음을 가져야 합니다:

| 필드 | 규칙 |
|------|------|
| 이름 | 필수, 그룹 간에 고유해야 함 |
| 비율 | 1-99%, 정수만 |
| 메시지 | 필수 (대조군 제외) |

## 오류 메시지

유효성 검사가 실패하면 다음과 같은 메시지가 표시됩니다:

| 오류 | 의미 |
|------|------|
| "Campaign name required" | 이름을 입력하지 않음 |
| "Name cannot exceed 50 characters" | 이름이 너무 김 |
| "At least 1 conversion event required for A/B test" | Experiment Campaign에 전환 이벤트가 필요함 |
| "Group percentages must equal 100%" | Experiment 그룹 분할이 합산되지 않음 |
| "Maximum 1 control group allowed" | 대조군이 너무 많음 |
| "Duplicate event names not allowed" | 같은 이벤트 이름을 두 번 사용함 |
| "Cannot use notifly__ prefix" | 해당 이벤트 이름은 내부용으로 예약됨 |
