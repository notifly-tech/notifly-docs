## 개요

인앱 메시지는 사용자가 앱을 활발히 사용하는 동안 화면에 나타나는 알림입니다. 앱 외부에 나타나는 푸시 알림과 달리, 인앱 메시지는 앱이 열려 있을 때 표시됩니다. 프로모션, 공지사항, 사용자 안내에 적합합니다.

## 작동 방식

1. 사용자가 Notifly SDK가 설치된 앱을 엽니다
2. SDK가 대기 중인 인앱 메시지를 확인합니다
3. 트리거 조건이 충족되면 메시지가 나타납니다
4. 사용자가 팝업을 보고 상호작용할 수 있습니다
5. 사용자가 닫거나 액션을 취합니다

## 요구 사항

- 사용자가 앱을 열고 있음
- 앱에 Notifly SDK 연동
- 트리거 조건 충족 (User Journey의 경우)

## 메시지 콘텐츠

| 필드 | 설명 | 필수 |
|-------|-------------|----------|
| HTML URL | 호스팅된 HTML 콘텐츠 | 예 |
| Template name | 템플릿 식별자 | 예 |
| Modal properties | 표시 설정 | 예 |

## Modal 속성

메시지가 어떻게 나타나는지 제어:

| 속성 | 설명 |
|----------|-------------|
| Position | 화면 위치 (center, top, bottom) |
| Width | 너비 (뷰포트 퍼센트) |
| Height | 높이 (뷰포트 퍼센트) |
| Max height | 최대 높이 제한 |
| Background opacity | 팝업 뒤 어둡게 (0-1) |
| Border radius | 둥근 모서리 |
| Dismiss on CTA tap | 버튼 탭하면 닫기 |

## 트리거 (User Journey에서)

User Journey의 인앱 메시지는 특별한 트리거 옵션이 있습니다:

| 설정 | 설명 |
|---------|-------------|
| Delay seconds | 표시 전 대기 |
| Trigger duration | 트리거 대기 시간 |
| Triggering conditions | 메시지를 표시하는 이벤트 |
| Triggering event filters | 트리거 이벤트 필터 |

## 템플릿 시스템

인앱 메시지는 HTML 템플릿을 사용합니다:
- 사전 제작된 템플릿 제공
- 커스텀 HTML 지원
- 인터랙티브 요소 (버튼, 링크)

## 개인화

**인앱 메시지는 Liquid 템플릿 개인화를 지원하지 않습니다.**

인앱 메시지는 SDK에 전달되는 사전 렌더링된 HTML 템플릿을 사용합니다. Liquid 구문은 발송 시점에 처리되지 않습니다.

개인화된 인앱 경험을 위해 다음을 사용할 수 있습니다:
- Notifly 콘솔에서 정의된 템플릿 파라미터 사용
- User Journey 컨텍스트를 통해 데이터 전달
- HTML 템플릿에서 JavaScript로 클라이언트 측 사용자 데이터 가져오기

## 규칙

- 앱이 포그라운드에 있을 때만 표시
- 사용자가 앱을 열고 있어야 함
- 한 번에 하나의 메시지만 표시
- [피로도 관리](/ko/knowledge-base/rules/fatigue-management) 적용
- [제외 사용자](/ko/knowledge-base/rules/excluded-users)에서 비활성화 가능
- [발송 금지 시간](/ko/knowledge-base/rules/forbidden-timing) 준수

## 예외 사항

- 앱이 백그라운드에 있으면 메시지 대기
- 트리거 전에 사용자가 앱을 닫으면 메시지 손실 가능
- 템플릿 오류가 있으면 표시되지 않음

## 사용 사례

| 사용 사례 | 예시 |
|----------|---------|
| 환영 | 첫 로그인 후 인사말 표시 |
| 프로모션 | 세일 배너 표시 |
| 기능 발견 | 새 기능 강조 |
| 피드백 요청 | 구매 후 리뷰 요청 |
| 온보딩 | 새 사용자 단계별 안내 |
