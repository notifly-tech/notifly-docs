## 개요

알림톡은 사용자에게 중요한 정보를 보내는 카카오 채널입니다. 주문 업데이트, 배송 알림, 예약 알림과 같은 트랜잭션 메시지용입니다. 광고용이 아닙니다.

## 작동 방식

1. 카카오에서 메시지 템플릿을 생성하고 승인받습니다
2. 사용자가 전화번호를 제공합니다
3. 승인된 템플릿을 사용해 메시지를 보냅니다
4. 메시지가 사용자의 카카오톡 앱에 나타납니다
5. 발송 실패 시 SMS로 대체 발송 가능

## 요구 사항

- 사용자가 카카오에 등록된 전화번호를 보유
- 등록된 카카오 비즈니스 채널 보유
- 카카오에서 승인된 메시지 템플릿

## 메시지 콘텐츠

| 필드 | 설명 | 필수 |
|-------|-------------|----------|
| Template code | 승인된 템플릿 ID | 예 |
| Template params | 템플릿에 채울 값 | 예 |
| Failover SMS | 카카오 실패 시 백업 메시지 | 아니오 |

## 템플릿 유형

| 유형 | 설명 |
|------|-------------|
| Basic | 텍스트만 |
| Emphasize | 강조된 제목과 부제목 |
| Image | 이미지 포함 |
| Item List | 항목 목록 표시 |

## 템플릿 요소

템플릿에 포함될 수 있는 것:
- **텍스트 콘텐츠** - 본문 메시지
- **버튼** - 링크 또는 액션 (웹, 앱, 전화 등)
- **빠른 답장** - 한 번 탭으로 응답 옵션
- **이미지** - 사진 또는 로고

## 버튼 유형

| 유형 | 기능 |
|------|--------------|
| Web link | URL 열기 |
| App link | 앱 열기 |
| Bot keyword | 챗봇 트리거 |
| Message delivery | 미리 정의된 메시지 발송 |
| Phone call | 전화 걸기 |

## 개인화

카카오 알림톡은 [Liquid 템플릿](/ko/knowledge-base/concepts/liquid-templates) 개인화를 전체 지원합니다.

**지원되는 필드:**
- 템플릿 파라미터
- 대체 발송 SMS 텍스트

**예시:**
```liquid
Template param (order_number): {{event.order_id}}
Template param (delivery_date): {{event.estimated_date}}
Failover SMS: 주문번호 {{event.order_id}} 배송 예정일: {{event.estimated_date}}
```

**참고:** 템플릿 구조는 고정되어 있지만 (카카오에서 승인), 템플릿 파라미터 값은 Liquid로 개인화할 수 있습니다.

## SMS 대체 발송

카카오 발송이 실패하면 (사용자가 카카오를 사용하지 않거나 차단 등):
- 자동으로 백업 SMS 발송
- Campaign 생성 시 SMS 내용 정의

## 규칙

- 템플릿은 사용 전 승인받아야 함
- 광고 발송 불가 (대신 [친구톡](/ko/knowledge-base/channels/kakao-friendtalk) 사용)
- 사용자가 전화번호와 연결된 카카오를 사용해야 함
- [피로도 관리](/ko/knowledge-base/rules/fatigue-management) 적용
- [제외 사용자](/ko/knowledge-base/rules/excluded-users)에서 비활성화 가능

## 예외 사항

- 사용자가 카카오를 사용하지 않으면 SMS로 대체 발송 (구성된 경우)
- 콘텐츠가 가이드라인을 위반하면 템플릿 거부
- 일부 템플릿은 추가 확인 필요
