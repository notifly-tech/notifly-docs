## 개요

LINE은 주로 일본, 대만, 태국에서 2억 명 이상의 사용자를 보유한 메시징 앱입니다. LINE 공식 계정을 팔로우하는 사용자에게 메시지를 보낼 수 있습니다.

## 작동 방식

1. 사용자가 LINE 공식 계정을 친구로 추가합니다
2. 사용자의 LINE User ID가 기록됩니다
3. 메시지를 작성합니다
4. LINE의 API를 통해 메시지가 발송됩니다
5. 메시지가 사용자의 LINE 채팅에 나타납니다

## 요구 사항

- 사용자가 LINE 공식 계정을 팔로우
- 사용자의 LINE User ID 보유
- Notifly에 LINE 공식 계정 구성

## 메시지 유형

| 유형 | 설명 |
|------|-------------|
| Text | 단순 텍스트 메시지 |

현재 Notifly는 LINE을 통한 텍스트 메시지를 지원합니다. 시스템은 이모지와 멘션을 지원하는 textV2 형식을 사용합니다.

## 발송 유형

| 유형 | 설명 | 제한 |
|------|-------------|-------|
| Multicast | 한 번에 여러 사용자에게 발송 | 최대 500명 |

## 메시지 콘텐츠

| 필드 | 설명 | 필수 |
|-------|-------------|----------|
| Message type | 메시지 유형 (text) | 예 |
| Send type | 발송 방법 (multicast) | 예 |
| Text | 메시지 내용 | 예 |

## 팔로우 상태

사용자는 다양한 팔로우 상태를 가집니다:

| 상태 | 의미 |
|--------|---------------|
| Followed | 사용자가 친구임, 메시지 수신 가능 |
| Unfollowed | 사용자가 차단함, 메시지 수신 불가 |
| Unknown | 상태를 확인할 수 없음 |

## 개인화

**LINE은 현재 Liquid 템플릿 개인화를 지원하지 않습니다.**

LINE 메시지의 Liquid 구문은 처리되지 않으며 원시 템플릿 텍스트가 메시지에 그대로 나타납니다. 이는 향후 업데이트에서 해결될 수 있는 플랫폼 제한 사항입니다.

동적 콘텐츠의 경우 다음을 고려하세요:
- 개인화된 콘텐츠 링크가 있는 [푸시 알림](/ko/knowledge-base/channels/push-notification)
- 자체 LINE 연동을 트리거하는 [Webhook](/ko/knowledge-base/channels/webhook)

## 규칙

- 사용자가 LINE 계정을 팔로우해야 함
- 차단된 사용자는 메시지 수신 불가
- 메시지 제한은 LINE 요금제에 따라 다름
- [피로도 관리](/ko/knowledge-base/rules/fatigue-management) 적용
- [제외 사용자](/ko/knowledge-base/rules/excluded-users)에서 비활성화 가능

## 예외 사항

- 사용자가 언팔로우하면 메시지 건너뜀
- LINE User ID가 유효하지 않으면 발송 실패
- LINE 요금제에 따라 속도 제한이 적용될 수 있음

## 지역별 고려사항

LINE은 다음 지역에서 가장 인기가 있습니다:
- 일본
- 대만
- 태국
- 인도네시아

사용자가 주로 이 지역에 있다면 LINE 사용을 고려하세요.
