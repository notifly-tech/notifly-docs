## 개요

SMS(Short Message Service)는 전화번호로 보내는 문자 메시지입니다. 스마트폰이나 앱 없이도 문자를 받을 수 있는 모든 휴대폰에서 작동합니다.

## 작동 방식

1. 사용자가 전화번호를 제공합니다
2. 짧은 문자 메시지를 작성합니다
3. Notifly가 SMS 제공업체를 통해 발송합니다
4. 메시지가 사용자의 휴대폰에 문자로 도착합니다

## 요구 사항

- 사용자의 유효한 전화번호
- Notifly에 SMS 발송 기능 구성

## 메시지 콘텐츠

| 필드 | 설명 | 필수 |
|-------|-------------|----------|
| Message body | 텍스트 콘텐츠 | 예 |
| Message title | 제목 (MMS용) | 아니오 |
| Is ad | 광고로 표시 | 아니오 |

## 메시지 유형

| 유형 | 설명 | 길이 |
|------|-------------|--------|
| SMS | 짧은 텍스트만 | 최대 80자(한글) 또는 160자(영문) |
| LMS | 긴 텍스트 | 최대 2,000자 |
| MMS | 이미지 포함 | 최대 2,000자 + 이미지 |

## 광고 규칙

메시지가 광고인 경우:
- 광고로 표시해야 함 (`is_ad: true`)
- 발신자 정보 포함해야 함
- 수신 거부 안내 포함해야 함
- 한국: "(광고)" 접두사 필요

## 개인화

SMS는 [Liquid 템플릿](/ko/knowledge-base/concepts/liquid-templates) 개인화를 전체 지원합니다.

**지원되는 필드:**
- Message body
- Message title (MMS용)

**예시:**
```liquid
{{user.$name | default: "고객"}}님, 주문번호 {{event.order_id}} 배송이 시작되었습니다.
```

**중요:** 이모지가 개인화 콘텐츠에서 자동으로 제거됩니다. 일부 통신사에서 SMS 메시지의 이모지 인코딩 문제가 있기 때문입니다.

## 규칙

- 사용자가 유효한 전화번호를 가지고 있어야 함
- 국가 및 인코딩에 따라 문자 제한이 다름
- 광고 메시지는 법적 요구 사항이 있음
- [피로도 관리](/ko/knowledge-base/rules/fatigue-management) 적용
- [제외 사용자](/ko/knowledge-base/rules/excluded-users)에서 비활성화 가능
- 메시지당 비용 발생

## 예외 사항

- 유효하지 않은 전화번호는 발송 실패 유발
- 국제 번호는 특별한 처리가 필요할 수 있음
- 일부 번호는 마케팅 메시지를 차단함

## 문제 해결

### 일반적인 발송 문제

| 증상 | 원인 | 해결책 |
|---------|-------|----------|
| 유효하지 않은 번호 | 형식 오류 | 국가 코드 포함 (+82...) |
| 발송 불가 | 번호가 존재하지 않음 | 목록에서 제거 |
| 통신사 차단 | 콘텐츠 필터링됨 | 메시지 내용 조정 |
| 사용자 수신 거부 | STOP 회신함 | 수신 거부 존중 |

### 디버깅 체크리스트

- [ ] 전화번호에 국가 코드가 포함되어 있나요?
- [ ] 번호 형식이 유효한가요 (E.164)?
- [ ] 사용자가 SMS 수신 거부하지 않았나요?
- [ ] 메시지가 문자 제한 내인가요?
- [ ] 발신자가 등록되어 있나요 (필요한 경우)?

**전체 문제 해결 가이드:** [SMS 실패](/ko/knowledge-base/troubleshooting/channel-failures/sms)
