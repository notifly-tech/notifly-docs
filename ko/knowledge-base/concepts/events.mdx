## 개요

Event는 "앱 열기", "버튼 클릭", "구매 완료"처럼 사용자가 취하는 행동입니다. 앱에서 event 데이터를 Notifly로 전송하면, Notifly가 이를 사용해 campaign을 트리거하고, segment를 구성하고, 전환을 추적합니다.

## 작동 방식

1. **Event 추적** - 앱이 event 데이터를 Notifly로 전송합니다
2. **Event 사용** - Campaign 트리거, segment 필터링, 전환 추적에 활용합니다
3. **분석** - 시간에 따른 사용자 행동을 확인합니다

## Event 구조

모든 event는 다음을 포함합니다:

| 요소 | 설명 |
|------|------------|
| Event name | 무엇이 일어났는지 (예: "purchase_completed") |
| User ID | 누가 했는지 |
| Timestamp | 언제 일어났는지 |
| Parameters | 추가 세부 정보 (예: product_id, price) |

## Event 활용

### Campaign 트리거
사용자가 특정 행동을 할 때 메시지 발송:
- Event name 매칭 (예: "cart_abandoned")
- 선택 사항: event parameter로 필터링 (예: "cart_value > 50000")

### User Journey 트리거
Event 발생 시 저니 시작:
- Campaign과 동일하지만, 사용자가 다단계 흐름에 진입합니다

### Cancellation Event (취소 이벤트)
발송 전 campaign 중지:
- 사용자가 메시지를 불필요하게 만드는 행동을 했을 때
- 예: 사용자가 구매를 완료하면 "장바구니 이탈" 메시지 취소

### Segment 구성
Event 이력으로 사용자 필터링:
- 횟수 기반: "Event를 최소 X회 수행"
- 기간 기반: "최근 Y일 내 event를 X회 수행"

### 전환 추적
메시지 수신 후 사용자가 목표를 완료했는지 추적:
- Campaign이나 journey에 전환 event 설정
- 분석에서 전환율 확인

## Event 조건

유연한 방식으로 event를 매칭할 수 있습니다:

| 조건 | 예시 |
|-----------|---------|
| Equals | Event name = "purchase" |
| Does not equal | Event name != "test_event" |
| Starts with | Event name이 "product_"로 시작 |
| Contains | Event name에 "click" 포함 |
| Matches regex | Event name이 패턴과 일치 |

## Event Parameter 필터

Event 데이터를 기반으로 추가 조건 설정:

| 필터 | 예시 |
|--------|---------|
| Equals | price = 10000 |
| Greater than | cart_value > 50000 |
| Contains | product_name에 "shirt" 포함 |
| Is null | coupon_code가 비어있음 |

## 규칙

- Event name은 대소문자를 구분합니다
- Event는 반드시 사용자 식별자를 포함해야 합니다
- Parameter는 선택 사항이지만 필터링에 유용합니다
- Event는 저장되며 과거 조회에 사용할 수 있습니다
