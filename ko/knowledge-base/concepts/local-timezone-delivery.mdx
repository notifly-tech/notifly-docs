## 개요

모든 사람에게 오전 9시에 메시지를 보내면 (일부 사용자에게는 자정일 수 있음), 로컬 타임존 발송은 각 사용자의 위치에 맞게 발송 시간을 조정합니다. 서울에 있는 사용자는 서울 시간 오전 9시에 메시지를 받습니다. 뉴욕에 있는 사용자는 뉴욕 시간 오전 9시에 받습니다.

## 작동 방식

1. **발송 시간 설정** - 예를 들어, "오전 9:00"
2. **시스템이 각 사용자의 타임존 확인** - 이 사용자는 어디에 사나요?
3. **해당 타임존의 오전 9시에 메시지 발송** - 각 사용자가 적절한 현지 시간에 받습니다

## 타임존 우선순위

시스템은 다음 순서로 사용자의 타임존을 찾습니다:

| 우선순위 | 출처 | 설명 |
|----------|--------|-------------|
| 1 (최고) | 사용자 타임존 | 사용자 속성에 설정됨 |
| 2 | 기기 타임존 | 기기에서 감지됨 |
| 3 (대체) | 기본 타임존 | Asia/Seoul (KST) |

사용자 프로필에 타임존이 설정되어 있으면 그것이 먼저 사용됩니다. 없으면 시스템이 기기의 타임존을 사용합니다. 둘 다 없으면 한국 표준시(KST)가 기본값입니다.

## 설정

### User Journey 설정

User Journey를 생성할 때 타임존 동작을 구성할 수 있습니다:

| 설정 | 기능 |
|---------|--------------|
| Use local timezone | 각 사용자의 타임존 기반 발송 활성화 |
| Default timezone | 사용자 타임존이 알려지지 않을 때의 대체 타임존 |
| Specific timezone | 모든 사람에게 이 타임존 사용 (오버라이드) |

### Campaign 설정

예약된 campaign의 경우, 타임존으로 수신자를 필터링할 수 있습니다:

| 설정 | 기능 |
|---------|--------------|
| Timezone condition | 특정 타임존의 사용자에게만 발송 |

## 타임존 형식

타임존은 IANA 표준 형식을 사용합니다:

| 예시 | 위치 |
|---------|----------|
| Asia/Seoul | 서울, 한국 |
| America/New_York | 뉴욕, 미국 |
| Europe/London | 런던, 영국 |
| Asia/Tokyo | 도쿄, 일본 |
| America/Los_Angeles | 로스앤젤레스, 미국 |
| Europe/Paris | 파리, 프랑스 |

IANA 타임존 이름의 전체 목록은 [Wikipedia](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones)에서 확인할 수 있습니다.

## 규칙

- 타임존은 텍스트로 저장됩니다 ("Asia/Seoul" 같은)
- 타임존이 유효하지 않거나 누락되면 기본값이 사용됩니다
- Delay 노드는 구성 시점에 시간을 계산합니다 - 로컬 시간을 UTC로 변환합니다
- 시스템은 일광 절약 시간을 자동으로 처리합니다

## 제한 사항

### CSV 및 Cohort Segment

**중요:** 로컬 타임존 발송은 조건 기반 segment에서만 작동합니다.

| Segment 유형 | 로컬 타임존 작동 여부 |
|--------------|----------------------|
| 조건 기반 | ✅ 예 |
| CSV 업로드 | ❌ 아니오 |
| Cohort | ❌ 아니오 |

CSV나 Cohort segment를 사용하면 모든 사용자가 같은 시간에 메시지를 받습니다 (기본 타임존 기준).

### 이 제한이 존재하는 이유

CSV와 Cohort segment는 사용자 ID의 정적 목록입니다. 시스템이 발송 시점에 타임존별로 동적으로 필터링하거나 예약할 수 없습니다. 조건 기반 segment는 실시간으로 평가되어 사용자별 타임존 계산이 가능합니다.

## 일반적인 사용 사례

| 사용 사례 | 설정 방법 |
|----------|------------------|
| 아침 환영 이메일 | 로컬 타임존을 활성화하고 오전 9시 현지 시간에 발송 |
| 글로벌 제품 출시 | 특정 타임존을 사용해 전 세계 동기화 |
| 지역 campaign | 타임존 조건을 사용해 한 지역 타겟팅 |
| 발송 금지 시간 준수 | 로컬 발송 금지 시간을 위해 forbidden timing과 결합 |

## 문제 해결

### 사용자가 잘못된 시간에 메시지를 받는 경우

1. **타임존 설정 확인** - 사용자 속성에서 타임존 값 확인
2. **기기 타임존 확인** - 기기가 타임존을 보고했는지 확인
3. **Segment 유형 확인** - CSV/Cohort는 로컬 타임존을 지원하지 않음을 기억하세요
4. **타임존 형식 확인** - 유효한 IANA 형식이어야 합니다 ("KST"가 아닌 "Asia/Seoul")

### 일부 사용자가 메시지를 받지 못하는 경우

Campaign에서 타임존 조건을 사용하면, 일치하는 타임존이 없는 사용자는 제외됩니다. 확인 사항:
- 사용자에게 타임존 데이터가 있는지
- 조건에 의도한 모든 타임존이 포함되어 있는지
