## 개요

보안 설정은 Notifly 프로젝트를 보호하는 데 도움이 됩니다. 팀원이 할 수 있는 것, 추가 로그인 단계 요구, 프로젝트에 접근할 수 있는 컴퓨터를 제한할 수 있습니다.

---

## User Permission Management (유저 권한 관리)

### 기능

활성화되면 **admin** 멤버만 민감한 기능에 접근할 수 있습니다. 일반 멤버는 제한된 접근 권한을 갖습니다.

### 역할 작동 방식

| 역할 | 설정 방법 | 접근 가능 항목 |
|------|----------|---------------|
| **Admin** | 멤버 페이지에서 설정 | 모든 것 |
| **Member** | 기본 역할 | 제한됨 (아래 참조) |

### Admin만 할 수 있는 것

User Permission Management가 **활성화**되면 admin만 다음을 할 수 있습니다:
- **사용자** 섹션 보기 및 관리 (사용자 목록, segment, 제외 사용자)
- **결제** 정보 및 이력 접근
- **멤버** 관리 (초대, 제거, 역할 변경)
- **보안 설정** 변경

### 비활성화 시

모든 사람이 모든 기능에 동등하게 접근합니다. 모든 팀원이 사용자, 결제 정보를 보고 멤버를 관리할 수 있습니다.

### 기본값

기본적으로 활성화됨.

## Session Management (세션 관리)

### 기능

로그인 세션을 제어합니다. 활성화되면 사용자가 로그인 상태를 유지하는 방식에 추가 보안 제어가 추가됩니다.

### 활성화 시 기능

- 세션 모니터링 가능
- Admin이 활성 세션 볼 수 있음
- 세션 정책 적용 가능

### 사용 이유

- **보안** - 로그인 상태에 대한 더 나은 제어
- **규정 준수** - 일부 보안 정책에서 세션 관리 필요

### 기본값

비활성화됨. 사용자가 수동으로 로그아웃할 때까지 로그인 상태 유지.

---

## OTP-only Login (OTP 전용 로그인)

### 기능

모든 로그인에 일회용 비밀번호(OTP)를 요구합니다. 이는 모든 프로젝트 멤버에게 필수인 2단계 인증(2FA)입니다.

### 작동 방식

1. 사용자가 이메일과 비밀번호 입력
2. 사용자가 인증 앱을 통해 6자리 코드 받음
3. 사용자가 코드를 입력하여 로그인 완료
4. 코드는 30초마다 변경됨

### 사용 이유

- **더 강한 보안** - 비밀번호가 도난당해도 코드 없이는 로그인 불가
- **규정 준수** - 많은 보안 표준에서 2FA 요구
- **안심** - 민감한 고객 데이터가 있는 프로젝트에 중요

### 중요 참고사항

- 이 기능을 활성화하기 전에 모든 팀원이 계정 설정에서 OTP를 설정해야 함
- 멤버가 인증 앱 접근을 잃으면 지원팀에 연락 필요

### 기본값

비활성화됨. 사용자는 이메일과 비밀번호만으로 로그인 가능.

---

## Access Control (접근 제어)

### 기능

활성화되면 승인된 IP 주소에서만 콘솔 접근을 제한합니다. 다른 IP 주소에서 접근하려는 사람은 차단됩니다.

### 작동 방식

1. Access Control 활성화
2. 허용 IP 목록에 IP 주소 추가
3. 해당 IP 주소에서 연결하는 사용자만 프로젝트에 접근 가능

### 사용 이유

- **사무실 전용 접근** - 회사 사무실 네트워크에서만 접근 허용
- **추가 보안** - 자격증명이 도난당해도 공격자가 프로젝트에 접근 불가
- **규정 준수** - 일부 규정에서 IP 기반 접근 제한 요구

### 중요 참고사항

- 활성화 전에 자신의 IP 주소를 추가해야 함, 그렇지 않으면 본인이 잠김
- 허용 IP 목록은 Access Control이 활성화된 경우에만 작동
- 원격 근무 시 집 IP를 추가하거나 VPN 사용 필요

### 기본값

비활성화됨. 유효한 자격증명이 있는 사람은 어디서든 로그인 가능.

---

## Allowed IP List (허용 IP 목록)

### 기능

Access Control이 활성화되었을 때 프로젝트에 접근할 수 있는 IP 주소 목록입니다.

### 사용 방법

- IP 주소를 쉼표로 구분하여 입력
- 예시: `192.168.1.1, 10.0.0.1, 203.0.113.50`

### IP 주소 찾는 방법

1. Google에서 "what is my IP" 검색
2. 표시된 IP 주소 복사
3. 허용 IP 목록에 추가

### 중요 참고사항

- 이 설정은 **Access Control이 활성화된 경우에만** 의미가 있음
- IP 주소는 변경될 수 있음, 특히 가정용 인터넷 연결의 경우
- 유연성을 위해 여러 IP (사무실, 집, 백업) 추가
- 기본값 `127.0.0.1`은 localhost (로컬 테스트에만 유용)

### 기본값

`127.0.0.1` (localhost placeholder).

## 모범 사례

### 소규모 팀 (1-5명)

1. User Permission Management 활성화 유지
2. 최소 한 명을 admin으로 설정
3. 더 나은 보안을 위해 OTP 활성화 고려

### 대규모 팀

1. User Permission Management 활성화
2. 필요한 사람에게만 admin 역할 부여
3. 모든 멤버에게 OTP-only Login 활성화
4. 고정 사무실 위치가 있으면 Access Control 고려

### 엔터프라이즈 / 높은 보안 요구

1. 모든 보안 기능 활성화
2. 사무실 IP 주소로 Access Control 사용
3. 모든 사람에게 OTP 요구
4. 정기적으로 admin 접근 권한 검토

## 문제 해결

### "사용자 또는 결제 메뉴가 보이지 않음"

User Permission Management가 활성화되어 있고 admin 역할이 없습니다. 현재 admin에게 멤버 섹션에서 admin 접근 권한을 요청하세요.

### "Access Control 활성화 후 잠김"

Notifly 지원팀에 문의하세요. Access Control을 비활성화하여 IP 목록을 수정할 수 있도록 도와줄 수 있습니다.

### "팀원이 OTP로 로그인할 수 없음"

1. 인증 앱의 시간이 올바르게 동기화되어 있는지 확인
2. 계정 설정에서 OTP를 다시 설정해야 할 수 있음
3. 문제가 지속되면 지원팀에 문의
