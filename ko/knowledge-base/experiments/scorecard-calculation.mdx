## Scorecard란?

Scorecard는 A/B 테스트의 "성적표"입니다. "Variant B가 정말 Variant A보다 나은가, 아니면 차이가 우연일 뿐인가?"라는 질문에 답합니다.

## 중요한 이유

통계 없이는 이렇게 생각할 수 있습니다:
- "Variant B가 52개 전환, Variant A가 48개 전환. B가 이겼다!"

하지만 각각 100명의 사용자만 있다면, 4개 전환 차이는 쉽게 우연일 수 있습니다. Scorecard는 차이가 **실제**인지 알려줍니다.

## 계산 프로세스

### 1단계: 트리거된 사용자 식별

실험에 참여한 모든 사람 찾기:
- 각 variant를 받은 고유 사용자 수
- 이들이 "피험자"

| Variant | 사용자 |
|---------|--------|
| A (Control) | 5,000 |
| B (Treatment) | 5,000 |

### 2단계: 전환 수 세기

각 그룹에서 전환한 사용자 수:

| Variant | 사용자 | 전환 |
|---------|--------|------|
| A | 5,000 | 200 |
| B | 5,000 | 250 |

### 3단계: 지표 값 계산

지표 값 = 전환 ÷ 사용자

| Variant | 계산 | 비율 |
|---------|------|------|
| A | 200 ÷ 5,000 | 4.0% |
| B | 250 ÷ 5,000 | 5.0% |

### 4단계: 통계 테스트 실행

정규 분포를 사용한 Z-test로 variant 비교.

## Scorecard 출력

| 필드 | 의미 | 예시 |
|------|------|------|
| `metric_value_1` | Control (A) 전환율 | 0.04 (4%) |
| `metric_value_2` | Treatment (B) 전환율 | 0.05 (5%) |
| `diff_in_percentage` | % 차이 | 25% |
| `p_value` | 통계적 유의성 | 0.023 |
| `confidence_interval_95_lb` | 95% CI 하한 | 3% |
| `confidence_interval_95_ub` | 95% CI 상한 | 47% |
| `mde` | 최소 감지 가능 효과 | 15% |
| `n_subjects_1` | Control 사용자 | 5,000 |
| `n_subjects_2` | Treatment 사용자 | 5,000 |

## 숫자 이해하기

### 차이 백분율

Treatment가 Control보다 얼마나 나은가 (또는 나쁜가)?

```
diff_percentage = (Treatment - Control) / Control × 100
```

예시:
- Control: 4%
- Treatment: 5%
- 차이: (5 - 4) / 4 × 100 = **25%**

Treatment가 Control보다 25% 더 나음.

### P-Value

차이가 우연일 확률.

| P-Value | 해석 |
|---------|------|
| < 0.01 | 매우 확신 (99%) |
| < 0.05 | 확신 (95%) - 표준 임계값 |
| 0.05 - 0.10 | 어느 정도 확신 |
| > 0.10 | 확신 없음 - 우연일 수 있음 |

**낮을수록 좋음.** p-value 0.023은 차이가 우연일 확률이 2.3%뿐임을 의미합니다.

### 신뢰 구간 (95%)

실제 차이가 있을 것으로 보이는 범위.

예시: CI = [3%, 47%]
- 실제 차이가 3%에서 47% 사이일 것으로 95% 확신
- 0%가 이 범위에 없으므로 차이는 유의함

**핵심 통찰**: 신뢰 구간에 0%가 포함되면 차이가 실제가 아닐 수 있습니다.

### MDE (최소 감지 가능 효과)

테스트가 신뢰성 있게 감지할 수 있는 가장 작은 차이.

다음에 따라 달라짐:
- 샘플 크기 (더 많은 사용자 = 더 작은 MDE)
- 데이터의 분산
- 신뢰 수준 (일반적으로 95%)

**관찰된 차이가 MDE보다 작으면**, 확신하려면 더 많은 데이터가 필요합니다.

## 결과가 준비되는 시점

### 처리 일정

- Scorecard는 **3시간**마다 계산 (매시간이 아님)
- 일반 통계보다 더 오래 걸림
- 실험이 충분히 진행된 후 다시 확인

### 최소 요구사항

의미 있는 결과를 위해 충분한 데이터 필요:
- 두 variant 모두에서 최소한의 전환
- 통계 계산을 위한 충분한 사용자
- 너무 적으면 scorecard에 결론 불가로 표시

## 결과 해석

### 명확한 승자

| 지표 | 의미 |
|------|------|
| p-value < 0.05 | 통계적으로 유의함 |
| CI에 0 포함 안됨 | 차이가 실제 |
| 의미 있는 상승 | 구현할 가치 있음 |

**조치**: 승리한 variant 사용!

### 명확한 승자 없음

| 지표 | 의미 |
|------|------|
| p-value > 0.05 | 유의하지 않음 |
| CI에 0 포함됨 | 차이가 0일 수 있음 |
| 작은 상승 | 가치가 없을 수 있음 |

**조치**: 더 많은 데이터를 위해 테스트 계속하거나 Control 유지 (더 단순함).

### 부정적 결과

Treatment가 Control보다 **나쁨**:
- 음수 차이
- 유의한 p-value

**조치**: Treatment 사용하지 않기! Control 유지.

## 예시 워크스루

### 시나리오

세일 campaign의 두 이메일 제목 테스트:
- **Variant A**: "주말 세일: 20% 할인"
- **Variant B**: "플래시 세일: 이번 주말 큰 할인!"

### 결과

| 지표 | Variant A | Variant B |
|------|-----------|-----------|
| 발송 | 10,000 | 10,000 |
| 오픈 | 2,000 | 2,400 |
| 구매 | 100 | 130 |

### Scorecard

| 필드 | 값 |
|------|------|
| metric_value_1 (A) | 1.0% (100/10,000) |
| metric_value_2 (B) | 1.3% (130/10,000) |
| diff_in_percentage | 30% |
| p_value | 0.018 |
| confidence_interval_95 | [5%, 55%] |

### 해석

- Variant B가 30% 더 많은 구매
- p-value 0.018 < 0.05 = 유의함
- CI [5%, 55%]에 0 포함 안됨 = 실제 차이
- **결론**: Variant B가 승자!

## 자주 묻는 질문

### 테스트를 얼마나 오래 실행해야 하나요?

다음 중 하나가 될 때까지:
- 통계적 유의성 (p < 0.05), 또는
- 목표 상승을 감지하기에 충분한 데이터, 또는
- 미리 정해진 최대 시간

### 결과가 왔다 갔다 하면?

테스트 초기에는 정상입니다. 더 많은 데이터를 기다리세요. 샘플 크기가 커지면 결과가 안정됩니다.

### 결과가 좋아 보이면 일찍 중단해도 되나요?

권장하지 않습니다. 조기 중단은 잘못된 결론으로 이어질 수 있습니다. 테스트가 완료될 때까지 실행하세요.
