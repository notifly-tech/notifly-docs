## A

### Abort Message
조건이 충족되면 메시지 발송을 중단하는 Liquid 템플릿 태그입니다. 조건부 발송 로직에 사용됩니다. [Liquid 템플릿](/ko/knowledge-base/concepts/liquid-templates) 참조.

### Alimtalk (알림톡)
주문 업데이트나 예약 알림 같은 중요한 정보를 보내기 위한 카카오 메시지 유형입니다. 광고용이 아닙니다.

## B

### Billing Cycle (결제 주기)
결제 사이의 기간으로, 보통 한 달입니다. 무료 메시지 할당량은 각 주기가 시작될 때 초기화됩니다.

## C

### Campaign (캠페인)
사용자에게 보내고자 하는 메시지입니다. 한 번, 예약으로, 또는 특정 이벤트 발생 시 보낼 수 있습니다.

### Channel (채널)
메시지를 보내는 방법입니다. 예: 푸시 알림, 이메일, SMS.

### Connected Content
메시지 렌더링 중 외부 API에서 데이터를 가져오는 Liquid 템플릿 기능입니다. 15초 타임아웃과 1분 캐시가 있습니다. [Liquid 템플릿](/ko/knowledge-base/concepts/liquid-templates) 참조.

### Cohort (코호트)
특정 시점에 생성된 저장된 사용자 그룹입니다.

## D

### Device (기기)
사용자에게 속한 휴대폰, 태블릿 또는 컴퓨터입니다. 한 사용자가 여러 기기를 가질 수 있습니다.

### Device Token (기기 토큰)
특정 기기에 푸시 알림을 보낼 수 있게 해주는 특수 코드입니다.

## E

### Event (이벤트)
"버튼 클릭"이나 "구매 완료"처럼 사용자가 앱에서 하는 행동입니다. 이벤트는 메시지를 트리거할 수 있습니다.

### Excluded User (제외 사용자)
특정 유형의 메시지 수신을 거부한 사용자입니다.

### External User ID
사용자를 위한 고유 ID (시스템에서의 계정 ID 같은 것)입니다.

## F

### Fatigue Management (피로도 관리)
하나의 캠페인이 특정 기간 동안 한 사용자에게 보낼 수 있는 메시지 수를 제한하는 규칙입니다.

### Forbidden Timing (발송 금지 시간)
메시지를 보내지 않아야 하는 시간 (예: 늦은 밤)입니다.

### Friendtalk (친구톡)
마케팅 및 프로모션을 위한 카카오 메시지 유형입니다. 사용자가 카카오 채널의 친구여야 합니다.

## G

### Global Frequency Limit (글로벌 발송 빈도 제한)
모든 캠페인에서 한 사용자가 받을 수 있는 총 메시지 수를 제한하는 규칙입니다.

## I

### In-App Message (인앱 메시지)
사용자가 앱을 사용하는 동안 모바일 앱 내에 나타나는 팝업입니다.

### In-Web Message (인웹 메시지)
사용자가 브라우징하는 동안 웹사이트에 나타나는 팝업입니다.

## L

### lastCollectedAt
캠페인 통계가 마지막으로 업데이트된 시간을 나타내는 타임스탬프입니다. 데이터가 얼마나 최신인지 이해하는 데 사용됩니다.

### Local Timezone Delivery (로컬 타임존 발송)
각 사용자의 위치에 따라 적절한 시간에 메시지를 보내는 기능입니다. 오전 9시 메시지는 각 사용자의 현지 시간 오전 9시에 도착합니다. [로컬 타임존 발송](/ko/knowledge-base/concepts/local-timezone-delivery) 참조.

### LINE
일본, 대만, 태국에서 인기 있는 메시징 앱입니다.

### Liquid Template (Liquid 템플릿)
사용자 이름, 이벤트 데이터, 외부 API 응답 같은 동적 콘텐츠로 메시지를 개인화하는 템플릿 언어입니다. [Liquid 템플릿](/ko/knowledge-base/concepts/liquid-templates) 참조.

## M

### Message Cap (메시지 한도)
메시지당 비용을 지불하기 전에 매월 보낼 수 있는 무료 메시지 수입니다. 한도는 각 결제 주기마다 초기화됩니다.

## N

### Node (노드)
User Journey의 한 단계입니다. 대기, 메시지, 또는 결정 포인트가 될 수 있습니다.

### Notifly Device ID
Notifly가 각 기기에 생성하는 고유 ID입니다.

### Notifly User ID
Notifly가 각 사용자에게 생성하는 고유 ID입니다.

## O

### Operator (연산자)
"같음", "보다 큼", "포함"처럼 값을 비교하는 방법입니다.

## P

### Prorated Billing (비례 청구)
사용한 결제 주기의 일부만 청구하는 방식입니다. 요금제를 업그레이드할 때, 남은 날짜에 대한 가격 차이만 지불합니다.

### Push Notification (푸시 알림)
앱이 닫혀 있어도 사용자의 휴대폰에 나타나는 알림입니다.

## R

### Re-eligible Condition (재수신 조건)
사용자가 동일한 캠페인을 다시 받을 수 있는 조건 규칙입니다.

### Re-entry Policy (재진입 정책)
사용자가 동일한 User Journey를 다시 시작할 수 있는 조건 규칙입니다.

## S

### Segment (세그먼트)
규칙으로 정의된 사용자 그룹입니다 (예: "최근 7일 내 구매한 사용자").

### Skip (건너뜀)
규칙(피로도 관리나 발송 금지 시간 등) 때문에 메시지가 발송되지 않는 경우입니다.

## T

### Timezone (타임존)
"Asia/Seoul"이나 "America/New_York"처럼 IANA 형식으로 저장되는 지역의 현지 시간 설정입니다. [로컬 타임존 발송](/ko/knowledge-base/concepts/local-timezone-delivery)과 [발송 금지 시간](/ko/knowledge-base/rules/forbidden-timing)에 사용됩니다.

### Triggering Event (트리거 이벤트)
캠페인이나 User Journey를 시작하는 이벤트입니다.

## U

### User Journey (유저 여정)
여러 단계가 있는 자동화된 워크플로우입니다. 사용자는 자신의 행동에 따라 이동합니다.

### User Properties (사용자 속성)
사용자에 대한 정보로, 이름, 이메일, 선호도 등이 있습니다.

## V

### Variant (변형)
A/B 테스트에서 메시지의 한 버전입니다.

## W

### Webhook (웹훅)
특정 이벤트가 발생할 때 자체 서버로 데이터를 보내는 방법입니다.

### Web Push (웹 푸시)
사용자의 웹 브라우저에 나타나는 알림입니다.
