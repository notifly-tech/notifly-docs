## 개요

사용자가 메시지를 받고 싶지 않을 때 "제외"로 표시할 수 있습니다. 제외된 사용자는 수신 거부한 채널에서 메시지를 받지 않습니다. 이는 사용자 선호도와 GDPR과 같은 법적 요구사항을 존중합니다.

## 작동 방식

1. 사용자가 메시지를 원하지 않는다고 표시함 (수신 거부, 옵트아웃 등)
2. Notifly에서 제외로 표시함
3. Campaign이 실행될 때 제외 사용자가 필터링됨
4. 메시지가 `excluded_user_filter` 사유로 건너뛰어짐

## 제외 사용자 설정

각 제외 사용자는 채널별 설정이 있습니다:

| 채널 | 필드 |
|------|------|
| 이메일 | `email` |
| SMS | `text_message` |
| 푸시 알림 | `push_notification` |
| 웹 푸시 | `web_push_notification` |
| 인앱 메시지 | `in_app_message` |
| 인웹 메시지 | `in_web_message` |
| 카카오 알림톡 | `kakao_alimtalk` |
| 카카오 친구톡 | `kakao_friendtalk` |
| 브랜드 메시지 | `brand_message` |
| LINE | `line` |
| Cafe24 앱 푸시 | `cafe24_app_push_notification` |

각 필드는 `true` (제외) 또는 `false` (제외 아님)입니다.

## 사용자 정보

| 필드 | 설명 |
|------|------|
| Notifly User ID | 어떤 사용자가 제외되었는지 |
| Description | 제외 이유 (선택적 메모) |
| Created at | 제외가 추가된 시점 |
| Updated at | 제외가 마지막으로 변경된 시점 |

## 채널별 제외

사용자는 특정 채널만 수신 거부할 수 있습니다:
- 이메일은 제외되지만 푸시는 계속 수신
- 마케팅(친구톡)은 제외되지만 정보성(알림톡)은 허용
- 모든 채널 제외

## 메시지 건너뛰기 시점

메시지가 건너뛰어지는 경우 (`excluded_user_filter` 건너뛰기 유형):
- 사용자가 제외 사용자 목록에 있음
- 그리고 특정 채널이 제외로 표시됨

## 제외 관리

### 제외 추가
- 사용자가 이메일의 수신 거부 링크 클릭
- 사용자가 앱 설정에서 옵트아웃
- API 또는 대시보드를 통해 수동 추가
- CSV에서 가져오기

### 제외 제거
- 사용자가 다시 수신 동의
- API 또는 대시보드를 통해 수동 제거
- 사용자가 재구독

## 법적 요구사항

제외는 다음을 준수하는 데 도움이 됩니다:
- **GDPR** (유럽) - 마케팅 반대권
- **CAN-SPAM** (미국) - 이메일 수신 거부 요구사항
- **KPIPA** (한국) - 개인정보 보호

## 모범 사례

1. **즉시 수신 거부 반영** - 제외를 지연하지 않음
2. **쉬운 수신 거부** - 명확한 수신 거부 링크
3. **이유 기록** - description 필드 사용
4. **세밀한 제어 제공** - 사용자가 채널을 선택할 수 있도록
5. **정기적 정리** - 유효하지 않은 항목 제거

## 규칙

- 메시지 발송 전 제외 확인
- 채널별 설정으로 세밀한 제어 가능
- Campaign에서 재정의 불가
- 법적 준수를 위해 반드시 존중해야 함

## 예외 사항

- 트랜잭션 메시지는 다른 규칙이 있을 수 있음 (법적 예외)
- Webhook 채널은 제외 사용자를 확인하지 않음 (사용자 메시지가 아니므로)
- 테스트 모드는 화이트리스트 사용자에 대해 우회할 수 있음
