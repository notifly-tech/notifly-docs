## 개요

피로도 관리는 Campaign 수준의 제한입니다. 하나의 Campaign이 특정 기간 동안 동일한 사용자에게 너무 많은 메시지를 보내는 것을 방지합니다. 예를 들어, "장바구니 이탈 Campaign을 사용자당 주 3회 이상 보내지 않음."

## 작동 방식

1. Campaign에 재적격 조건이 설정됨
2. 사용자가 Campaign에서 메시지를 수신함
3. 시스템이 이 노출을 기록함
4. Campaign이 다시 발송하려 할 때 시스템이 횟수를 확인함
5. 제한에 도달하면 메시지가 건너뛰어짐

## 재적격 조건

재적격 조건은 다음을 정의합니다:

| 필드 | 의미 |
|------|------|
| Value | 시간 윈도우 크기 |
| Unit | 시간 단위 (h/d/w/m) |
| Max count | 이 윈도우 내 최대 메시지 수 |

## 시간 윈도우

| 단위 | 의미 | 기간 |
|------|------|------|
| h | 시간 | 1시간 = 3,600초 |
| d | 일 | 1일 = 24시간 |
| w | 주 | 7일 |
| m | 월 | 30일 |

## 예시

| 설정 | 의미 |
|------|------|
| 하루 1회 | 사용자가 이 Campaign을 24시간마다 최대 1번 수신 |
| 주 3회 | 사용자가 이 Campaign을 7일마다 최대 3번 수신 |
| 월 1회, 최대 5회 | 월 최대 1회, 총 5회 이상 절대 수신 불가 |

## 최대 횟수 제한

롤링 시간 윈도우 외에 절대 최대값을 설정할 수 있습니다:
- `max_count: 5`는 사용자가 이 Campaign에서 총 5개 이상의 메시지를 절대 받지 않음을 의미
- 시간 윈도우가 더 많이 허용하더라도 최대 횟수가 우선됨

## 메시지 건너뛰기 시점

메시지가 건너뛰어지는 경우 (`fatigue_management` 건너뛰기 유형):
- 사용자가 시간 윈도우 내에 이미 제한에 도달한 경우
- 또는 사용자가 최대 횟수에 도달한 경우 (설정된 경우)

## 글로벌 발송 빈도 제한과의 차이

| 피로도 관리 | 글로벌 발송 빈도 제한 |
|-------------|----------------------|
| Campaign별 제한 | 계정 전체 제한 |
| "이 Campaign만" | "모든 Campaign 합계" |
| 각 Campaign에 설정 | 프로젝트 설정에서 설정 |
| Campaign별로 독립 | 모든 Campaign에서 공유 |

## 규칙

- 이벤트 트리거 Campaign에만 적용
- 노출 횟수는 사용자별, Campaign별로 계산
- 시간 윈도우는 롤링 방식 (달력 기준 아님)
- 개별 메시지에서 재정의 불가

## 예외 사항

- 예약 Campaign은 피로도 관리를 사용하지 않음 (고정 시간에 실행)
- 테스트 모드 메시지는 제한에 포함되지 않음
- 사용자가 Campaign을 한 번도 받지 않았다면 피로도 관리 통과
