## 개요

글로벌 발송 빈도 제한은 계정 전체 설정입니다. 모든 Campaign을 합산하여 사용자가 받을 수 있는 메시지 수를 제한합니다. 예를 들어, "어떤 사용자도 하루에 5개 이상의 메시지를 받지 않도록 함."

## 작동 방식

1. 계정에 글로벌 발송 빈도 제한이 구성됨
2. 어떤 Campaign이든 사용자에게 발송하려 할 때 시스템이 최근 메시지 수를 계산
3. 사용자가 제한에 도달했거나 초과하면 메시지가 건너뛰어짐
4. 모든 채널을 계산하거나 채널별로 필터링 가능

## 글로벌 발송 빈도 제한 설정

각 제한은 다음을 포함합니다:

| 필드 | 의미 |
|------|------|
| ID | 이 제한의 고유 식별자 |
| Value | 시간 윈도우 크기 |
| Unit | 시간 단위 (h/d/w/m) |
| Max count | 이 윈도우 내 최대 메시지 수 |
| Channels | 적용할 채널 (선택사항) |

## 시간 윈도우

| 단위 | 의미 | 기간 |
|------|------|------|
| h | 시간 | 1시간 = 3,600,000ms |
| d | 일 | 24시간 |
| w | 주 | 7일 |
| m | 월 | 30일 |

## 예시

| 설정 | 의미 |
|------|------|
| 하루 3회 | 사용자가 하루에 최대 3개 메시지 수신 (모든 Campaign) |
| 주 10회 | 사용자가 주당 최대 10개 메시지 수신 |
| 하루 5회, 푸시만 | 사용자가 하루에 최대 5개 푸시 알림 수신 |

## 채널 필터링

특정 채널에만 적용되는 제한을 설정할 수 있습니다:
- `channels`가 비어있거나 설정되지 않은 경우: 모든 채널에 적용
- `channels`에 값이 있는 경우: 해당 채널만 계산/제한

## 다중 제한

여러 글로벌 발송 빈도 제한을 동시에 활성화할 수 있습니다:
- 하루 3회 그리고 주 10회
- 사용자가 메시지를 받으려면 모든 제한 미만이어야 함
- 어떤 제한이든 초과하면 메시지가 건너뛰어짐

## 우회 옵션

Campaign이 글로벌 발송 빈도 제한을 무시하도록 설정할 수 있습니다:
- Campaign에서 `ignore_global_frequency_capping: true` 설정
- 신중하게 사용 - 사용자 피로도로 이어질 수 있음
- 중요한 트랜잭션 메시지에 적합

## 메시지 건너뛰기 시점

메시지가 건너뛰어지는 경우 (`global_frequency_limit_filter` 건너뛰기 유형):
- 사용자가 활성 글로벌 발송 빈도 제한에 도달한 경우
- 제한이 Campaign의 채널에 적용되는 경우

## 피로도 관리와의 차이

| 글로벌 발송 빈도 제한 | 피로도 관리 |
|----------------------|-------------|
| 계정 전체 | Campaign별 |
| 모든 Campaign 합산 | 이 Campaign만 |
| 프로젝트 설정에서 설정 | 각 Campaign에 설정 |
| 공유 횟수 | 독립 횟수 |

## 규칙

- 모든 채널 (또는 필터링된 채널)에 적용
- 모든 Campaign을 함께 계산
- 어떤 활성 제한도 초과할 수 없음
- Campaign별로 우회 가능 (필요한 경우)

## 건너뛰기 데이터

건너뛰어질 때 건너뛰기 데이터에 포함되는 정보:
- 어떤 제한이 초과되었는지
- 현재 횟수 대 제한

## 예외 사항

- `ignore_global_frequency_capping`이 설정된 Campaign은 이 확인을 우회함
- Webhook 채널은 다르게 동작할 수 있음
- 테스트 메시지는 제한에 포함되지 않을 수 있음
