## Campaign 저장 안됨

### 폼의 유효성 검사 오류

**증상:** 저장 시도 시 빨간색 오류 메시지 표시

**일반적인 원인 및 해결:**

| 오류 | 원인 | 해결책 |
|------|------|--------|
| "Campaign name required" | 이름 필드 비어있음 | Campaign 이름 입력 |
| "Name cannot exceed 50 characters" | 이름이 너무 김 | 이름 줄이기 |
| "Percentages must equal 100%" | A/B 테스트 비율이 맞지 않음 | 비율 합계를 100%로 조정 |
| "At least 2 groups required" | A/B 테스트에 그룹 더 필요 | 다른 변형 그룹 추가 |
| "At least 1 conversion event required" | 실험에 목표 누락 | 전환 이벤트 추가 |

**전체 오류 목록:** [캠페인 오류 - UI 유효성 검사](/ko/knowledge-base/campaigns/errors.md#ui-validation-errors)

---

### 동시성 오류

**증상:** "Campaign was updated by another user"

**원인:** 편집 중 다른 사람이 campaign을 수정함

**해결:**
1. 페이지 새로고침
2. 변경 사항 다시 작성
3. 다시 저장
4. 팁: 동시 편집을 피하기 위해 팀과 조율

---

### 폼이 로드되지 않음

**증상:** Campaign 에디터가 빈 화면이거나 로드 안됨

**해결:**
1. 페이지 새로고침
2. 브라우저 캐시 삭제
3. 다른 브라우저 시도
4. 인터넷 연결 확인

---

## Campaign 활성화 안됨

### 필수 필드 누락

**증상:** 활성화 버튼 비활성화 또는 활성화 시 오류

**확인:**
- [ ] Campaign에 이름이 있음
- [ ] 최소 하나의 메시지가 설정됨
- [ ] Segment가 선택 또는 정의됨
- [ ] 일정 또는 트리거가 설정됨
- [ ] 채널 발신자가 설정됨

---

### 채널이 설정되지 않음

**증상:** "Missing required sender information"

**원인:** 프로젝트에 채널이 설정되지 않음

**해결:**
1. 프로젝트 설정으로 이동
2. 채널 설정으로 이동
3. 발신자 설정 완료 (API 키, 인증된 발신자 등)
4. Campaign으로 돌아와서 다시 시도

---

### 유효하지 않은 Segment

**증상:** 현재 segment 설정으로 활성화 불가

**확인:**
- Segment에 유효한 조건이 있나?
- 참조된 사용자 속성이 올바르게 철자되었나?
- Segment 유형이 campaign 유형과 호환되나?

**알려진 제한:**
- In-App/In-Web campaign은 segment 조건에서 개인화를 사용할 수 없음
- user-ID 없는 CSV segment는 csv.* 변수만 사용 가능

---

### 일정이 과거

**증상:** 예약 campaign이 활성화되지 않음

**원인:** 예약 시간이 이미 지남

**해결:**
1. 일정을 미래 시간으로 업데이트
2. 저장하고 활성화
3. 즉시 발송의 경우, 가능하다면 "Send Now" 사용

---

## Campaign 발송 안됨

### Campaign이 "Active"지만 메시지 발송 안됨

**트리거 유형별 확인:**

| 트리거 유형 | 확인 사항 |
|------------|----------|
| 예약 | 예약 시간이 지났는지 확인 |
| 이벤트 | SDK/API가 이벤트를 보내는지, 이벤트 이름 일치 확인 |
| API | API 호출이 올바르게 이루어지는지 확인 |

**참조:** 발송 수준 문제는 [메시지가 발송되지 않음](/ko/knowledge-base/troubleshooting/message-not-sending)

---

### Segment가 0명 반환

**증상:** Campaign 활성화되었지만 "0 users targeted"

**확인:**
1. Campaign 에디터에서 segment 미리보기
2. 조건이 너무 제한적이지 않은지 확인
3. AND vs OR 조합 확인

**일반적인 원인:**
- 속성 이름 오타 (대소문자 구분)
- 모든 사람을 제외하는 날짜 범위
- 서로 모순되는 조건

**해결:** [세그먼트 검증](/ko/knowledge-base/troubleshooting/segment-verification)

---

### 이벤트 트리거 불일치

**증상:** 이벤트 발생하지만 campaign 트리거 안됨

**확인:**
1. 이벤트 이름이 정확히 일치 (대소문자 구분)
2. 이벤트가 올바른 프로젝트로 전송되고 있음
3. 추가 트리거 조건 (있는 경우) 충족

**디버그:**
- 이벤트 디버거로 수신 이벤트 확인
- Campaign의 이벤트 이름과 SDK가 보내는 것 비교
- Campaign이 속성 필터를 사용하면 이벤트 속성 확인

## 잘못된 타겟팅

### 너무 많은 사용자가 수신

**증상:** 받지 말아야 할 사용자에게 메시지 발송됨

**확인:**
- Segment가 조건에 AND 대신 OR 사용?
- 조건이 그룹 내에서 제대로 중첩?
- 기본 "모든 사용자" 폴백이 있나?

**해결:**
1. Segment 조건 신중히 검토
2. Segment 미리보기로 일치하는 사람 확인
3. 먼저 소규모 그룹으로 테스트

---

### 너무 적은 사용자가 수신

**증상:** 예상한 사용자가 메시지를 받지 못함

**확인:**
- OR이 필요한데 AND 사용?
- 조건이 너무 제한적?
- 발송 규칙에 의해 필터링?

**일반적인 원인:**
- 많은 AND 조건 결합으로 거의 모든 사람 제외
- 속성 값이 일치하지 않음 (대소문자 구분 문자열)
- 피로도/빈도 제한에 의해 필터링

---

### 동일 사용자에게 중복 메시지

**증상:** 일부 사용자가 같은 메시지를 여러 번 수신

**확인:**
- Campaign이 중복 허용으로 설정?
- 이벤트 트리거의 경우: 이벤트가 여러 번 발생?
- Segment 설정에서 "allow duplicate user" 활성화?

**해결:**
- Fatigue management로 사용자당 메시지 제한
- 앱에서 중복 이벤트 발생 수정
- Segment 설정에서 "allow duplicate user" 비활성화

---

## A/B 테스트 문제

### 불균등 분배

**증상:** 그룹이 예상 비율을 받지 못함

**원인:** 작은 샘플 크기에서의 무작위 분배

**설명:** 작은 숫자에서는 실제 분배가 비율과 다를 수 있음. 큰 샘플에서 균등해짐.

**예시:** 10명의 50/50 분할은 6/4 또는 7/3을 보일 수 있음. 1000명에서는 500/500에 가까워짐.

---

### 전환 추적 안됨

**증상:** A/B 테스트에 전환이 표시되지 않음

**확인:**
1. 전환 이벤트가 올바르게 정의?
2. 앱에서 전환 이벤트를 보내고 있나?
3. 전환 창이 적절한가?

**해결:**
- 이벤트 이름이 정확히 일치하는지 확인
- SDK가 전환 이벤트를 보내는지 확인
- 행동에 시간이 걸리면 전환 창 증가

---

### 승자 결정 불가

**증상:** 테스트 완료되었지만 명확한 승자 없음

**원인:**
- 사용자 부족 (통계적 유의성)
- 변형 간 성능이 비슷
- 전환 창이 너무 짧음

**해결:**
- 더 오래 또는 더 많은 사용자로 테스트 실행
- 더 다른 변형 테스트
- 전환 창 연장

## API 트리거 문제

### "Campaign is not configured as API triggered"

**원인:** Campaign이 예약 또는 이벤트 트리거이고, API가 아님

**해결:**
1. Campaign 편집
2. 트리거 유형을 "API"로 변경
3. 저장하고 활성화
4. API 호출 다시 시도

---

### "Campaign is not active"

**원인:** Campaign이 Draft 또는 Terminated 상태

**해결:**
1. Campaign 활성화
2. API 호출 다시 시도

---

### "Too many recipients"

**원인:** API 호출에서 1,000명 이상의 수신자 전송

**해결:**
- 1,000명 이하 배치로 분할
- 여러 API 호출 실행

---

### "Invalid recipients list"

**원인:** 수신자 데이터가 잘못된 형식

**확인:**
- JSON 배열이 올바르게 포맷?
- 수신자 객체에 필수 필드 있음?
- 수신자 유형이 이 채널에 유효?

**유형별 필수 필드:**
| 수신자 유형 | 필수 필드 |
|------------|----------|
| user-id | `user_id` |
| phone-number | `phone_number` |
| email | `email` |
| line-user-id | `line_user_id` |

**참조:** [캠페인 오류 - 수신자 오류](/ko/knowledge-base/campaigns/errors.md#recipient-errors)

---

### 인증 오류 (401)

**원인:** 유효하지 않거나 누락된 API 자격증명

**해결:**
1. API 키가 올바른지 확인
2. API 키가 만료되지 않았는지 확인
3. API 키가 이 프로젝트에 권한이 있는지 확인
4. Authorization 헤더 형식 확인

## 상태 전환 문제

상태 전환에 대한 자세한 내용은 [캠페인 라이프사이클](/ko/knowledge-base/campaigns/lifecycle)을 참조하세요.

| 문제 | 원인 | 해결 |
|------|------|------|
| Draft에서 멈춤 | 필수 필드 누락 또는 권한 없음 | 유효성 검사 오류 확인 |
| Active 편집 불가 | 활성 상태에서 일부 필드 잠김 | 일시 중지 후 편집 |
| 예기치 않게 종료 | 수동 종료, 예약 종료, 오류 | 새 campaign 생성 필요 |

## 진단 체크리스트

### 활성화 전

- [ ] Campaign 이름 설정됨
- [ ] 채널 선택 및 설정됨
- [ ] 메시지 내용 완료
- [ ] Segment/타겟팅 정의됨
- [ ] 일정 또는 트리거 설정됨
- [ ] A/B 테스트: 그룹 합계 100%, 전환 이벤트 설정됨

### Campaign 발송 안될 때

- [ ] 상태가 "Active"
- [ ] 일정 시간 지남 (예약의 경우)
- [ ] 이벤트 수신 중 (이벤트 트리거의 경우)
- [ ] API 호출 올바름 (API 트리거의 경우)
- [ ] Segment에 일치하는 사용자 있음
- [ ] 발송 로그에서 skip 사유 확인
