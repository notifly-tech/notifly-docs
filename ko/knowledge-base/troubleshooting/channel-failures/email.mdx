## 빠른 참조

| 증상 | 가능한 원인 | 해결책 |
|------|------------|--------|
| 잘못된 주소 | 오타 또는 가짜 이메일 | 이메일 형식 확인 |
| Hard bounce | 주소가 존재하지 않음 | 목록에서 제거 |
| Soft bounce | 메일함 가득 참, 일시적 문제 | 재시도 또는 대기 |
| 스팸 폴더 | 발신자 평판 낮음 | 평판 개선 |
| 이메일 없음 | 프로필에 이메일 없음 | 이메일 수집 |

## 일반적인 장애

### 잘못된 이메일 주소

**의미:** 이메일 주소 형식이 잘못되었거나 존재하지 않습니다.

**잘못된 유형:**
- 형식 오류: `john@` (도메인 누락)
- 오타: `john@gmial.com`
- 존재하지 않음: `random123@company.com`

**확인 방법:**
- 발송 상태가 "Invalid" 또는 "Hard Bounce"로 표시
- 반송 메시지에 "user unknown" 또는 "no such user" 언급

**해결 방법:**
- 저장 전 이메일 형식 검증
- 사용자 프로필에서 잘못된 이메일 표시
- 알려진 잘못된 주소로 발송하지 않기
- 이메일 수집 시 더블 옵트인 고려

---

### Hard Bounce

**의미:** 이메일이 영구적으로 거부되었습니다.

**일반적인 원인:**
- 이메일 주소가 존재하지 않음
- 도메인이 존재하지 않음
- 메일함이 영구적으로 비활성화됨

**처리 방법:**
- 이메일을 무효로 표시
- 이 주소로 발송 중지
- 재시도하지 않음 - 작동하지 않음

---

### Soft Bounce

**의미:** 일시적인 발송 실패입니다.

**일반적인 원인:**
- 메일함이 가득 참
- 서버 일시적 불가
- 메시지 크기 초과
- 수신 서버 바쁨

**처리 방법:**
- 나중에 재시도
- 여러 번 soft bounce 후 hard bounce로 처리
- 지속적으로 실패하면 메시지 크기 확인

---

### 전달되었지만 받지 못함

**의미:** 시스템에서는 전달됨으로 표시되지만 사용자가 받지 못했다고 함.

**확인 위치:**
1. 스팸/정크 폴더
2. 프로모션 탭 (Gmail)
3. 기타 폴더 (필터 설정된 경우)

**일반적인 원인:**
- 스팸 필터링
- 사용자 필터/규칙 설정
- 이메일 클라이언트가 특정 메시지 숨김

---

### 스팸으로 분류됨

**발생 이유:**
- 발신자 평판 낮음
- 콘텐츠가 스팸 필터 트리거
- 인증 누락 (SPF, DKIM, DMARC)
- 이미지가 너무 많고 텍스트가 적음
- 제목/본문에 스팸성 단어

**확인 방법:**
- 사용자에게 스팸 폴더 확인 요청
- 이메일 테스트 도구로 스팸 점수 확인

**해결 방법:**
- 적절한 인증 설정 (SPF, DKIM, DMARC)
- 콘텐츠 개선 (텍스트/이미지 균형)
- 스팸 트리거 단어 피하기
- 발신자 평판 점진적 구축
- 인식 가능한 발신자 이름 사용

---

### 인증 실패

**SPF (Sender Policy Framework):**
- 발신자가 도메인에서 발송할 수 있는지 확인
- 실패 = 스팸 위험 증가

**DKIM (DomainKeys Identified Mail):**
- 메시지 무결성을 확인하는 암호화 서명
- 실패 = 메시지 변조 가능성

**DMARC (Domain-based Message Authentication):**
- SPF/DKIM 실패 시 수신자에게 처리 방법 지시
- 적절히 설정하면 전달률 향상

**확인 방법:**
- 이메일 헤더에서 인증 결과 확인
- 이메일 테스트 도구 사용
- DNS 레코드 설정 확인

**해결 방법:**
1. 발송 도메인에 SPF 레코드 설정
2. Notifly에서 DKIM 서명 활성화
3. DMARC 레코드 추가
4. 도움이 필요하면 지원팀 연락

## 발신자 평판 문제

### 발신자 평판이란?

이메일 제공업체가 수신자가 이메일과 상호작용하는 방식을 추적합니다:
- 오픈율
- 클릭율
- 스팸 신고
- 반송
- 수신 거부

낮은 지표 = 낮은 평판 = 더 많은 스팸 필터링

---

### 낮은 평판 징후

- 스팸 폴더 배치 증가
- 시간이 지남에 따라 오픈율 감소
- 발송 실패 증가
- 블록리스트에 등록됨

---

### 개선 방법

| 해야 할 것 | 하지 말아야 할 것 |
|-----------|------------------|
| 참여하는 사용자에게 발송 | 모든 사람에게 발송 |
| 목록 정기적 정리 | 잘못된 주소 유지 |
| 수신 거부 즉시 처리 | 수신 거부 어렵게 만들기 |
| 일정한 발송량 유지 | 발송량 급증 |
| 가치 있는 콘텐츠 | 프로모션 콘텐츠만 |

## 설정 문제

### 발신자 미인증

**의미:** 발신 이메일 주소가 이메일 제공업체에서 인증되지 않음.

**증상:** 모든 이메일 실패, 또는 스팸으로 분류됨.

**해결 방법:**
1. 프로젝트 설정 > 이메일로 이동
2. 발신자 이메일 주소 인증
3. 인증 단계 따르기 (보통 링크 클릭)

---

### 잘못된 발신자 설정

**확인 항목:**
- Campaign에 발신자 이메일 설정됨
- 회신 주소가 유효함
- 발신자 이름 설정됨

---

## 콘텐츠 문제

### 이메일 크기 초과

**증상:** "message too large"로 soft bounce

**제한:**
- 대부분의 제공업체: 총 10-25 MB
- 권장: 100KB 이하가 최적

**해결:**
- 이미지 압축
- 이미지를 외부에서 호스팅하고 링크
- 불필요한 첨부 파일 제거

---

### 렌더링 문제

**증상:** 이메일이 깨지거나 콘텐츠 누락

**일반적인 원인:**
- 이메일 클라이언트에서 CSS 지원 안됨
- 이미지 기본적으로 차단됨
- HTML 호환성 문제

**해결:**
- 이메일 안전 HTML/CSS 사용
- 이미지에 alt 텍스트 포함
- 여러 이메일 클라이언트에서 테스트

- [ ] 이메일 콘텐츠가 스팸 검사를 통과하는가?
- [ ] 수신 거부 링크가 포함되어 있는가?

## 오류 메시지 참조

| 오류 | 의미 | 조치 |
|------|------|------|
| Invalid recipient | 이메일 형식 잘못됨 | 이메일 주소 수정 |
| User unknown | 계정이 존재하지 않음 | 무효로 표시 |
| Mailbox full | 사용자 받은 편지함 가득 참 | 나중에 재시도 |
| Message rejected | 서버가 메시지 거부 | 콘텐츠/평판 확인 |
| Connection timeout | 서버 연결 불가 | 나중에 재시도 |
| SPF fail | 인증 실패 | SPF 설정 |
| DKIM fail | 서명 무효 | DKIM 설정 확인 |
| Rate limited | 이메일 너무 많이 발송 | 발송 속도 줄이기 |
| Blocklisted | IP/도메인 블록리스트됨 | 제공업체 연락 |

---

## 반송 처리

### 모범 사례

1. **Hard bounce:** 즉시 무효로 표시, 다시 발송하지 않음
2. **Soft bounce:** 2-3회 재시도 후 무효로 표시
3. **스팸 신고:** 즉시 처리, 블록리스트 고려
4. **수신 거부:** 즉시 처리

### 목록 정리

- Hard bounce 자동 제거
- Soft bounce 추적, 임계값 초과 시 제거
- 열지 않는 사용자 제거 (X개월 후)
- 수신 거부 쉽게 만들기
