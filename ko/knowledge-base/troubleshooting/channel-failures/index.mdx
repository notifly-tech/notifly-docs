## 개요

메시지가 발송 규칙에 의해 건너뛰어지지 않았지만 여전히 전달되지 않는 경우, 문제는 보통 채널별 문제입니다. 이 섹션은 각 채널의 일반적인 장애를 다룹니다.

## 채널 가이드

| 채널 | 일반적인 문제 |
|------|--------------|
| [푸시 알림](/ko/knowledge-base/troubleshooting/channel-failures/push-notification) | 잘못된 토큰, 권한 거부, 만료된 토큰 |
| [이메일](/ko/knowledge-base/troubleshooting/channel-failures/email) | 잘못된 주소, 반송, 스팸 필터링 |
| [SMS](/ko/knowledge-base/troubleshooting/channel-failures/sms) | 잘못된 번호, 통신사 차단, 수신 거부 |
| [카카오](/ko/knowledge-base/troubleshooting/channel-failures/kakao) | 템플릿 문제, 친구 상태, 채널 제한 |
| [LINE](/ko/knowledge-base/troubleshooting/channel-failures/line) | 잘못된 사용자 ID, 차단된 계정, 할당량 제한 |

## 증상별 빠른 진단

### "Delivery Failed" 오류

| 채널 | 가능한 원인 | 확인 항목 |
|------|------------|----------|
| 푸시 | 잘못된/만료된 토큰 | 디바이스 토큰 유효성 |
| 이메일 | 잘못된 주소 | 이메일 형식과 존재 여부 |
| SMS | 잘못된 번호 | 전화번호 형식 |
| 카카오 | 템플릿 미승인 | 템플릿 상태 |
| LINE | 사용자 미연결 | LINE 사용자 ID 유효성 |

### "Delivery Succeeded"지만 사용자가 수신 못함

| 채널 | 가능한 원인 | 확인 항목 |
|------|------------|----------|
| 푸시 | 앱 알림 비활성화 | 디바이스 설정 |
| 이메일 | 스팸으로 분류 | 스팸 폴더, 발신자 평판 |
| SMS | 통신사 필터링 | 통신사 제한 |
| 카카오 | 메시지 숨김 | 카카오 채팅 설정 |
| LINE | 알림 음소거 | LINE 알림 설정 |

## 공통 사전 확인

채널별 문제로 들어가기 전에 다음 공통 요구사항을 확인하세요:

### 1. 프로젝트에 채널 설정됨

- [ ] 프로젝트 설정에서 채널 활성화됨
- [ ] 발신자/자격증명 설정됨
- [ ] API 키/토큰 유효하고 만료 안됨

### 2. 사용자가 필수 식별자 보유

| 채널 | 필수 |
|------|-----|
| 푸시 | 디바이스 토큰 |
| 이메일 | 이메일 주소 |
| SMS | 전화번호 |
| 카카오 알림톡 | 전화번호 |
| 카카오 친구톡 | 카카오 연결 + 친구 상태 |
| LINE | LINE 사용자 ID |

### 3. 사용자 미제외

- [ ] 사용자가 이 채널의 제외 사용자 목록에 없음
- [ ] 사용자가 이 채널 구독 취소 안함
- [ ] 사용자가 이 채널에서 무효로 표시 안됨

## 발송 상태 참조

### 성공 상태

| 상태 | 의미 |
|------|------|
| Sent | 메시지가 제공업체로 전송됨 |
| Delivered | 디바이스에 전달 확인됨 |

### 실패 상태

| 상태 | 의미 |
|------|------|
| Failed | 제공업체로 전송 불가 |
| Bounced | 수신 시스템에서 거부됨 |
| Undeliverable | 수신자에게 도달 불가 |
| Invalid | 수신자 주소/토큰 무효 |

### 대기 상태

| 상태 | 의미 |
|------|------|
| Queued | 전송 대기 중 |
| Pending | 전송됨, 확인 대기 중 |

## 에스컬레이션 시점

다음 경우 지원팀에 에스컬레이션:
- 제공업체 API가 예기치 않은 오류 반환
- 발송률이 갑자기 떨어짐
- 같은 오류가 모든 사용자에게 영향
- 명백한 원인 수정 후에도 오류 지속

**에스컬레이션 전 수집:**
- 채널 유형
- 오류 메시지/코드
- 영향받는 사용자 샘플
- 문제 시작 시점
- 설정의 최근 변경 사항
