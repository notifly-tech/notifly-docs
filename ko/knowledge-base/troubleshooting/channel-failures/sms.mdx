## 빠른 참조

| 증상 | 가능한 원인 | 해결책 |
|------|------------|--------|
| 잘못된 번호 | 형식 오류 또는 가짜 | 전화번호 형식 검증 |
| 전달 불가 | 번호가 존재하지 않음 | 목록에서 제거 |
| 통신사 차단 | 메시지 필터링됨 | 콘텐츠 조정 |
| 수신 거부 | 사용자가 SMS 수신 거부 | 설정 존중 |
| 번호 없음 | 프로필에 전화번호 없음 | 전화번호 수집 |

---

## 일반적인 장애

### 잘못된 전화번호

**의미:** 전화번호 형식이 잘못되었거나 번호가 존재하지 않습니다.

**일반적인 형식 문제:**
- 국가 코드 누락
- 자릿수 오류
- 번호에 문자 포함
- 공백 또는 특수문자

**올바른 형식:**
| 국가 | 형식 | 예시 |
|------|------|------|
| 미국/캐나다 | +1XXXXXXXXXX | +12025551234 |
| 한국 | +82XXXXXXXXXX | +821012345678 |
| 일본 | +81XXXXXXXXXX | +818012345678 |

**해결 방법:**
- 국가 코드와 함께 번호 저장
- 저장 전 형식 검증
- 잘못된 번호 목록에서 제거

---

### 연결 불가

**의미:** 번호는 존재하지만 메시지 전달이 안 됩니다.

**가능한 원인:**
- 전화기 꺼짐
- 서비스 지역 외
- 번호 해지됨
- 통신사 네트워크 문제

**처리 방법:**
- 지연 후 한 번 재시도
- 지속되면 연결 불가로 표시

---

### 통신사 필터링

**의미:** 이동통신사가 메시지를 차단했습니다.

**통신사 필터링 이유:**
- 스팸처럼 보임
- 발신자로부터 대량 발송
- 차단된 키워드 포함
- 발신자가 제대로 등록되지 않음

**필터링 징후:**
- 미전달 비율 높음
- 발송 상태에 "Filtered" 또는 "Blocked" 표시
- 일부 통신사에서만 작동, 다른 곳에서 안됨

**해결 방법:**
- 통신사에 발신자 ID 등록 (필요한 경우)
- 스팸성 콘텐츠 피하기
- 메시지를 너무 빠르게 많이 보내지 않기
- 승인된 발신번호 사용

---

### 사용자 수신 거부

**의미:** 사용자가 STOP을 회신하거나 SMS 수신 거부했습니다.

**작동 방식:**
- 사용자가 번호로 "STOP" 문자
- 통신사가 수신 거부 등록
- 이후 메시지 차단됨

**처리 방법:**
- 수신 거부 즉시 존중
- 사용자를 SMS 제외로 표시
- 다시 발송 시도하지 않기
- 원하면 재수신 신청 방법 제공

---

## 발신자 문제

### 미등록 발신자

**의미:** 발신번호/ID가 제대로 등록되지 않았습니다.

**국가별 요구사항:**
| 국가 | 요구사항 |
|------|----------|
| 미국 | 10DLC 등록 또는 수신자 부담 전화 |
| 한국 | 등록된 사업자 번호 |
| 많은 국가 | 영숫자 발신자 ID 등록 |

**해결 방법:**
1. 국가 요구사항 확인
2. 통신사 또는 규제 기관에 발신자 등록
3. Notifly에서 등록된 발신자 설정

---

### 발신자 ID 표시 안됨

**의미:** 수신자가 브랜드명 대신 임의 번호를 봅니다.

**발생 이유:**
- 영숫자 발신자 ID 지원 안됨
- 브랜드 발신자 미등록
- 통신사가 발신자 ID 덮어씀

**참고:** 일부 국가는 영숫자 발신자 ID를 지원하지 않습니다.

---

## 콘텐츠 문제

### 메시지가 너무 김

**의미:** SMS가 문자 제한을 초과합니다.

**표준 SMS 제한:**
| 인코딩 | 세그먼트당 문자 수 |
|--------|-------------------|
| GSM-7 | 160자 |
| Unicode | 70자 |

**메시지 분할 시:**
- 긴 메시지는 여러 세그먼트로 분할
- 각 세그먼트 = 1 SMS 요금
- 일부 문자는 사용 가능 길이 감소

**해결 방법:**
- 메시지를 간결하게 유지
- 가능하면 특수문자 피하기
- 유니코드 문자 (이모지, 비라틴) 더 많은 공간 사용

---

### 차단된 콘텐츠

**의미:** 통신사가 콘텐츠 기반으로 메시지를 필터링했습니다.

**일반적으로 필터링되는 것:**
- URL (특히 단축 URL)
- 메시지 본문의 전화번호
- 금융/은행 키워드
- "무료" 또는 프로모션 문구
- 성인 콘텐츠

**해결 방법:**
- 차단된 키워드 제거 또는 교체
- 단축 URL 대신 브랜드 URL 사용
- 대량 발송 전 콘텐츠 테스트

---

## 국가별 문제

### 한국

**요구사항:**
- 등록된 발신번호 사용 필수
- 마케팅 SMS에 080 번호 필요
- 수신 거부 안내 포함 필수

**일반적인 문제:**
- 미등록 발신번호 거부됨
- 광고에 080 번호 누락
- KISA 규정 미준수

---

### 미국

**요구사항:**
- A2P(Application-to-Person)용 10DLC 등록
- 수신자 부담 전화번호 인증
- TCPA 준수 (동의 필요)

**일반적인 문제:**
- 미등록 번호에서 발송
- 통신사가 미등록 트래픽 필터링
- 동의 기록 누락

---

## 속도 제한

### 상황

**통신사 제한:**
- 각 통신사마다 처리량 제한
- 제한 초과 = 지연 또는 거부

**증상:**
- 일부 메시지 지연
- 일부 메시지가 "throttled"로 실패
- 적은 양에서는 작동, 많은 양에서 실패

**처리 방법:**
- 한 번에 보내지 않고 일정한 속도로 발송
- 가능한 경우 throttling 설정 사용
- 대형 campaign을 시간에 걸쳐 예약

## 디버깅 체크리스트

### 발송 전

- [ ] 사용자 프로필에 전화번호가 있는가?
- [ ] 번호에 국가 코드가 포함되어 있는가?
- [ ] 번호 형식이 유효한가?
- [ ] 사용자가 SMS 수신 거부하지 않았는가?
- [ ] 발신번호/ID가 등록되었는가?
- [ ] 메시지가 문자 제한 내인가?
- [ ] 콘텐츠가 SMS에 적절한가?

### 실패 후

- [ ] 발송 상태에 무엇이 표시되는가?
- [ ] 전화번호가 유효한가?
- [ ] 한 사용자인가 여러 사용자인가?
- [ ] 모든 실패가 같은 통신사인가?
- [ ] 콘텐츠에 필터링된 단어가 포함되어 있는가?
- [ ] 발송량이 속도 제한을 유발하는가?

## 오류 메시지 참조

| 오류 | 의미 | 조치 |
|------|------|------|
| Invalid number | 전화 형식 잘못됨 | 번호 형식 수정 |
| Unknown subscriber | 번호가 존재하지 않음 | 목록에서 제거 |
| Undeliverable | 전화에 연결 불가 | 재시도 또는 제거 |
| Filtered | 통신사가 메시지 차단 | 콘텐츠 확인 |
| Opted out | 사용자가 수신 거부 | 제외로 표시 |
| Rate limited | 메시지가 너무 많음 | 속도 줄이기 |
| Invalid sender | 발신자 ID 허용 안됨 | 발신자 등록 |
| Content blocked | 메시지 콘텐츠 차단됨 | 콘텐츠 수정 |

---

## 발송 상태 의미

| 상태 | 의미 |
|------|------|
| Sent | 통신사로 메시지 전송됨 |
| Delivered | 전화로 전달 확인됨 |
| Failed | 발송 불가 |
| Undelivered | 발송했지만 전달 안됨 |
| Pending | 전달 확인 대기 중 |

**참고:** 모든 통신사가 전달 확인을 제공하지는 않습니다.

---

## 모범 사례

### 더 나은 전달률

1. **목록 정리**
   - 저장 전 번호 검증
   - 잘못된 번호 제거
   - 수신 거부 즉시 처리

2. **제대로 등록**
   - 필요한 곳에 발신자 ID 등록
   - 통신사 등록 완료
   - 승인된 번호만 사용

3. **콘텐츠 주의**
   - 메시지를 짧게 유지
   - 스팸 트리거 피하기
   - 수신 거부 안내 포함

4. **책임감 있게 발송**
   - 너무 자주 발송하지 않기
   - 발송 금지 시간 존중
   - 대량 발송을 시간에 걸쳐 분산
