## 사용자 미진입

### Journey 상태

**먼저 확인:** Journey가 "Running" 상태인가?

| 상태 | 사용자 진입 가능? |
|------|-----------------|
| Draft | 불가 |
| Running | 가능 |
| Paused | 불가 (기존 사용자는 계속) |
| Stopped | 불가 |

**해결:** 새 사용자가 진입하려면 상태를 "Running"으로 변경.

---

### 테스트 모드 활성화

**증상:** Journey가 실행 중이지만 일부 사용자만 진입

**원인:** 테스트 모드가 화이트리스트 사용자로 진입 제한

**확인:**
1. Journey 설정 열기
2. "Testing Mode" 토글 찾기
3. 화이트리스트 확인

**해결:**
- 테스트 사용자를 화이트리스트에 추가, 또는
- 프로덕션에서는 테스트 모드 비활성화

---

### 진입 트리거 불일치

**이벤트 트리거 Journey의 경우:**

| 확인 항목 | 확인 방법 |
|----------|----------|
| 이벤트 이름 정확히 일치? | Journey 트리거 vs SDK 이벤트 비교 (대소문자 구분) |
| 이벤트가 전송되고 있나? | 이벤트 디버거로 확인 |
| 이벤트 속성 일치? | Journey에 속성 필터가 있는지 확인 |

**예약 Journey의 경우:**

| 확인 항목 | 확인 방법 |
|----------|----------|
| 예약 시간 지남? | Journey 일정 설정 확인 |
| 타임존 올바름? | 타임존 설정 확인 |
| 예약 시간에 journey가 실행 중이었나? | Journey 상태 이력 확인 |

**API 트리거 Journey의 경우:**

| 확인 항목 | 확인 방법 |
|----------|----------|
| API 호출이 이루어지고 있나? | API 로그 확인 |
| 올바른 엔드포인트와 페이로드? | API 구현 확인 |
| 인증 유효? | API 응답 코드 확인 |

---

### 진입 Segment 필터링

**증상:** 진입 트리거 발생하지만 사용자 미진입

**원인:** 사용자가 진입 segment에 맞지 않음

**확인:**
1. Journey 진입에 어떤 segment가 설정?
2. 사용자가 해당 segment에 맞나?
3. Segment 미리보기로 자격 있는 사람 확인

**해결:**
- 사용자가 필요한 속성을 가지고 있는지 확인
- 너무 제한적이면 segment 조건 조정
- Segment에 맞는 것을 아는 사용자로 테스트

---

### 재진입 제한

**증상:** 사용자가 진입 이벤트를 트리거했지만 다시 진입 안함

**원인:** Journey가 재진입을 제한할 수 있음

**확인:**
1. 사용자가 이 journey를 이전에 완료했나?
2. 재진입 설정은 어떤가?
3. 쿨다운 기간이 있나?

**재진입 설정:**
| 설정 | 의미 |
|------|------|
| 재진입 불가 | 사용자가 한 번만 통과 가능 |
| 재진입 허용 | 완료 후 다시 진입 가능 |
| 쿨다운 기간 | 재진입 전 X일 대기 필요 |

---

### 사용자가 이미 Journey에 있음

**증상:** 사용자가 기준에 맞지만 진입 안함

**원인:** 사용자가 이미 이 journey에 있음

**확인:** 사용자의 현재 journey 위치 조회

**참고:** 사용자는 일반적으로 journey에서 한 번에 한 위치에만 있을 수 있습니다.

---

## Journey에서 사용자 멈춤

### Delay 노드에서 멈춤

**증상:** 사용자가 delay 노드에 예상보다 오래 머무름

**확인:**
1. Delay 기간은 얼마인가?
2. 사용자가 언제 delay에 진입했나?
3. Delay가 절대 또는 상대 시간인가?

**Delay 유형:**
| 유형 | 동작 |
|------|------|
| 대기 기간 | 노드 진입 후 X시간/일 대기 |
| 시간까지 대기 | 특정 시간까지 대기 |
| 날짜까지 대기 | 특정 날짜까지 대기 |

**일반적인 문제:**
- Delay가 예상보다 김 (단위 확인: 시간 vs 일)
- 시간까지 대기가 오늘 이미 지남 (내일까지 대기)
- 타임존 혼란

---

### 이벤트 분기에서 멈춤

**증상:** 사용자가 이벤트 분기 노드를 지나지 않음

**원인:** 사용자가 예상 이벤트를 수행하지 않음

**확인:**
1. 분기가 어떤 이벤트를 확인하나?
2. 타임아웃이 설정되어 있나?
3. 사용자가 일치하는 이벤트를 수행했나?

**이벤트 분기 작동 방식:**
- 사용자가 일치하는 이벤트를 수행할 때까지 노드에서 대기
- 타임아웃이 설정되면, 시간 만료 시 타임아웃 경로로 이동
- 타임아웃이 없으면 무기한 대기

**해결:**
- 이벤트 이름이 정확히 일치하는지 확인
- 행동하지 않는 사용자를 위한 타임아웃 경로 추가
- SDK가 이벤트를 올바르게 보내는지 확인

---

### 메시지 노드에서 멈춤 (진행 안함)

**증상:** 사용자가 메시지 노드에 있지만 앞으로 이동 안함

**가능한 원인:**
1. 메시지 발송이 아직 대기 중
2. 메시지 실패했지만 실패 경로 미설정
3. 노드 처리 시스템 문제

**확인:**
- 이 사용자에 대한 발송 기록이 있나?
- 발송 상태는?
- 메시지 노드에 성공과 실패 경로 모두 있나?

---

## 메시지 노드 문제

### 메시지 미발송

**증상:** 사용자가 메시지 노드를 통과했지만 메시지 전달 안됨

**의사결정 트리:**
```
Journey 노드에서 메시지 미발송?
        │
        ▼
이 사용자의 발송 로그 확인
        │
    발견? ──NO──► 메시지 시도 안됐을 수 있음
        │              Journey 흐름과 사용자 위치 확인
       YES
        │
        ▼
메시지가 건너뛰어졌나? ──YES──► Skip 사유 확인
        │                      (Campaign skip과 동일)
       NO
        │
        ▼
채널 요구사항과 사용자 데이터 확인
```

**모든 skip 사유 적용:** [메시지가 발송되지 않음 - Skip 사유](/ko/knowledge-base/troubleshooting/message-not-sending.md#skip-사유-설명)

---

### 메시지 발송됐지만 사용자가 실패 경로

**증상:** 메시지가 전달됐지만 사용자가 실패 경로로 진행

**가능한 원인:**
- 발송 확인됐지만 응답이 실패 표시
- 발송 확인 대기 중 타임아웃
- 채널이 성공 반환했지만 나중에 반송

**확인:**
- 로그의 발송 상태
- 채널별 발송 리포트
- 반송/불만 기록

---

### 메시지 전 Delay 작동 안함

**증상:** Delay 무시하고 메시지가 즉시 발송

**확인:**
1. 메시지 노드 전에 delay 노드가 있나?
2. Delay가 올바르게 설정?
3. 노드가 올바른 순서로 연결?

**해결:** Journey 에디터에서 노드 연결 확인

---

## 분기 문제

### 이벤트 분기가 이벤트 감지 못함

**증상:** 사용자가 이벤트를 수행했지만 이벤트 분기로 안 감

**확인:**
1. 이벤트 이름이 정확히 일치 (대소문자 구분)
2. 이벤트 타이밍이 대기 창 내
3. 이벤트 속성이 조건과 일치 (설정된 경우)

**중요한 타이밍 참고:**
- 이벤트는 사용자가 분기 노드에 도달한 후에 발생해야 함
- 노드에 도달하기 전 이벤트는 집계 안됨

---

### Segment 분기 잘못된 경로

**증상:** 사용자가 segment 기반으로 예상치 못한 분기로 감

**확인:**
1. 분기 시점에 사용자의 속성은?
2. Segment 조건이 올바른가?
3. 평가 순서가 예상대로인가?

**Segment 분기 평가 방식:**
- 사용자가 노드에 도달하는 순간 조건 확인
- 현재 사용자 속성 사용 (journey 진입 시점 아님)
- 순서대로 분기 평가, 첫 일치 경로 선택

**일반적인 문제:**
- Journey 진입 후 사용자 속성 변경
- 여러 분기가 일치할 수 있음, 첫 번째 선택
- "Else" 분기가 어떤 조건도 안 맞는 사용자 잡음

---

### 모든 사용자가 "Else" 분기로

**증상:** 정의된 분기 조건에 아무도 안 맞음

**확인:**
- 조건 필드 이름이 올바른가?
- 값이 사용자 데이터 형식과 일치?
- 비교 연산자가 올바른가?

**디버그:**
- 특정 사용자의 속성 확인
- 분기 조건과 비교
- 오타나 형식 불일치 찾기

---

## 예기치 않은 탈출

### Exit 조건 트리거됨

**증상:** 사용자가 "Conditional Exit" 노드를 통해 탈출

**원인:** 사용자가 exit 조건 충족

**확인:**
1. 어떤 exit 이벤트가 설정?
2. 사용자가 해당 이벤트 수행?
3. Exit 이벤트 타이밍은?

**Exit 조건 작동 방식:**
- Journey의 주요 지점에서 확인
- 사용자가 exit 이벤트 수행하면 journey 이탈
- 용도: 구매, 구독 취소, 목표 달성

---

### 비정상 탈출

**증상:** 사용자가 "Abnormal Exit" 노드를 통해 탈출

**가능한 원인:**
1. 노드 처리 오류
2. Journey 타임아웃 초과
3. 노드 설정 무효
4. 시스템 오류

**확인:**
- Journey 오류 로그
- 노드 설정 오류
- Journey 타임아웃 설정

---

### 사용자가 Journey에서 사라짐

**증상:** Journey에서 사용자를 찾을 수 없음, 탈출 기록 없음

**가능한 원인:**
- 사용자가 journey 정상 완료
- Journey가 중지/리셋됨
- 사용자 데이터 삭제
- 추적 문제

**확인:**
- Journey 완료 기록
- Journey 상태 변경 이력
- 시스템의 사용자 존재

---

## Journey 상태 문제

### Journey가 시작 안됨

**증상:** 상태를 "Running"으로 변경 불가

**확인:**
- Journey가 유효한가 (모든 필수 노드 연결)?
- 유효성 검사 오류가 있나?
- Journey 실행 권한이 있나?

**유효한 journey 필수 요소:**
- 시작 노드
- 탈출 노드로 가는 최소 하나의 완전한 경로
- 모든 노드 올바르게 연결
- 메시지 노드에 콘텐츠 설정

---

### Journey가 예기치 않게 일시 중지

**확인:**
- 누가 수동으로 일시 중지?
- 일시 중지를 트리거한 오류?
- Journey 상태 변경 로그 확인

---

### Journey가 중지되고 재시작 안됨

**참고:** 중지된 journey는 재시작 불가

**복구 옵션:**
1. Journey의 새 버전 생성
2. Journey 복제 후 복사본 시작
3. 중지된 journey의 사용자는 수동 처리 필요할 수 있음

---

## Journey 테스트

### 효과적인 테스트 방법

1. **테스트 모드 활성화**
   - 화이트리스트에 본인 추가
   - 화이트리스트 사용자만 진입 가능

2. **진입 트리거**
   - 진입 이벤트 수행 (또는 일정 대기)
   - Journey에 진입했는지 확인

3. **노드 통과 진행**
   - Delay: 테스트 중 짧은 기간 사용
   - 분기: 각 경로 테스트

4. **각 단계에서 확인**
   - 노드가 처리되었는지 확인
   - 메시지가 발송되었는지 확인
   - 올바른 경로 선택 확인

### 일반적인 테스트 실수

| 실수 | 문제 | 해결 |
|------|------|------|
| 테스트 모드 꺼짐 | 실제 사용자 영향 | 항상 테스트 모드 사용 |
| 긴 delay | 테스트 빨리 완료 못함 | 테스트용 짧은 delay 사용 |
| 잘못된 이벤트 이름 | 진입 트리거 안됨 | 이벤트 이름 재확인 |
| 화이트리스트 잊음 | Journey 진입 불가 | 화이트리스트에 본인 추가 |

## 진단 체크리스트

### Journey 작동 안함 체크리스트

**진입 문제:**
- [ ] Journey 상태가 "Running"
- [ ] 테스트 모드 꺼짐 (또는 사용자가 화이트리스트)
- [ ] 진입 트리거 일치 (이벤트 이름, 일정 시간)
- [ ] 사용자가 진입 segment에 맞음
- [ ] 사용자가 이미 journey에 없음
- [ ] 재진입 허용됨 (해당되는 경우)

**진행 문제:**
- [ ] 노드가 올바르게 연결
- [ ] Delay 기간이 예상대로
- [ ] 이벤트 분기 이벤트가 정확히 일치
- [ ] Segment 조건이 올바름
- [ ] 필요한 곳에 타임아웃 경로 존재

**메시지 문제:**
- [ ] 메시지 콘텐츠 설정됨
- [ ] 채널 요구사항 충족
- [ ] 사용자가 제외/필터링 안됨
- [ ] 발송 규칙이 차단 안함

**탈출 문제:**
- [ ] Exit 조건이 의도된 것
- [ ] Exit 이벤트가 사용자 행동과 일치
- [ ] 예기치 않은 비정상 탈출 없음
