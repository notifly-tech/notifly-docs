## 딜레이 설정

딜레이 노드에서는 일정 시간 / 가장 가까운 시간 / 특정 날짜까지 딜레이를 설정할 수 있습니다. 딜레이 노드에서 대기하는 동안 이탈 조건을 만족한다면, 딜레이 노트에서 이탈로 기록됩니다.

딜레이 시간/일시를 설정하는 방법은 2가지입니다.

- **고정값 딜레이**: 특정 기간 또는 일시를 고정값으로 설정
- **동적 딜레이**: 이벤트 기반 진입 조건의 이벤트 파라미터 값

### 일정 시간

설정한 시간만큼 대기합니다. 예를 들어, '12시간 30분'으로 설정된 경우 딜레이 노드에 진입한 시간부터 12시간 30분 이후에 다음 노드로 진행합니다.

### 가장 가까운 시간

다가오는 특정 시간까지 대기하도록 설정할 수 있습니다. 예를 들어 '오후 3시'로 설정된 딜레이 노드에 7월 23일 오후 6시에 진입한다면, 다가오는 오후 3시인 7월 24일 오후 3시에 다음 노드로 진행합니다. 만약 이틀 뒤 다가오는 시간으로 설정하고자 하는 경우, 해당 딜레이 노드 앞에 또다른 딜레이 노드로 '2일 동안' 대기하도록 설정할 수 있습니다.

### 특정 날짜까지

설정한 특정 날짜 시간까지 대기합니다. 설정된 시간 이후에 노드에 진입한다면 유저 여정에서 자동으로 이탈됩니다.

**동적 딜레이 사용** 토글을 활성화하면 딜레이 기간/일시를 개인화 할 수 있습니다.

<Check>
  동적 딜레이는 \*\*이벤트 기반 진입 \*\*시에만 사용할 수 있습니다.
</Check>

<Note>
  개인화 하려는 딜레이 기간 혹은 일시 정보가 진입 이벤트의 특정 파라미터로 포함되어 있어야 합니다.
</Note>

### 구성 요소

- **일정 기간 / 가장 가까운 미래 / 특정 시점까지** : 딜레이 옵션
- **이벤트 파라미터 키** : 개인화 하기 위한 정보값을 갖고 있는 이벤트 파라미터명
- **기본값 사용** : 이벤트 파라미터 값이 유효하지 않거나 null일 때 대체할 수 있는 고정값

### 일정 시간

설정된 시간만큼 대기합니다. \
**※** **파라미터 값은 초 단위의 숫자여야 합니다** (예: 600 = 10분동안 대기함)

### 가장 가까운 시간

돌아오는 특정 시간까지 대기합니다.\
**※ 파라미터 값은 ISO 8601 시간 형식이어야 합니다** (예: 14:30+09:00 = 가장 가까운 KST 14:30분까지 대기함)

### 특정 날짜까지

설정한 특정 날짜 시간까지 대기합니다. \
**※ 파라미터 값은 ISO 8601 datetime 형식이어야 합니다** (예: 2024-01-15T14:30:00+09:00 = KST 2024년 1월 15일 14:30까지 대기함)

## FAQ

<AccordionGroup>
  <Accordion title="동적 딜레이에서 파라미터 값이 유효하지 않을 때는 어떻게 되나요?">
    만일 '기본값 사용'이 활성화 되어있었다면 설정된 기본값만큼 딜레이가 될 것입니다. 만일 기본값을 세팅하지 않으셨다면 '진행 불가로 인한 이탈' 처리됩니다.
  </Accordion>
  <Accordion title="동적 딜레이에서 '특정 날짜까지' 조건으로 과거 시점 값을 주입하면 어떻게 되나요?">
    특정 날짜까지의 시점이 과거라면 '진행 불가로 인한 이탈' 처리됩니다. 만일 기본값을 세팅하셨더라도 먼저 검사하는 파라미터값이 과거 시점이기 때문에 마찬가지로 '진행 불가로 인한 이탈' 처리됩니다.
  </Accordion>
</AccordionGroup>
